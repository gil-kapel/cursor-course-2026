# שיעור 2.13 (ElevenLabs / עברית)
---

## 00:00 – 01:30

גם Agent מעולה וגם Skill טוב לא יעזרו אם פתיחת המשימה חלשה. כשהפתיח עמום, הסוכן מנחש.

## 01:30 – 03:30

תבנית פשוטה לפתיחת כל משימה: תפקיד. מטרה. קבצים עם `@`. פלט רצוי. מגבלות. "סיימנו כש..."

זה נשמע קטן, אבל זו כמעט כל התורה. לא "תבנה לי משהו" — אלא פתיח שאומר למודל בדיוק מי הוא, מה הקלט, מה הפלט, ומתי המשימה גמורה.

## 03:30 – 05:00

אותו עיקרון עובד ל-PRD, Build, Review, Debug. לכל משימה יש הקשר אחר. ולכן: **Task חדש = Context חדש**. אל תגררו שיחה ישנה למשימה חדשה.

## 05:00 – 06:00

תרגיל: קחו משימה אחת מהמודול — PRD, Build או Review — וכתבו לה פתיח חד ב-4-5 שורות. אחר כך הריצו את אותו Agent פעם עם פתיח חלש ופעם עם פתיח חד. תראו את ההבדל.

שמרו לעצמכם 2-3 פתיחים כאלה — זה ה-playbook הראשון שלכם. במודול הבא עוברים ל-MCP וכלים חיים — ושם prompt חד חשוב אפילו יותר.
