# Script Master: B04 Git as Save Points (GUI Only)

## Metadata

- Lesson ID: B04
- Title: Git כרשת ביטחון (נקודות שמירה)
- Target duration: 08:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

להציג Git כמכונת זמן / save points; commit ראשון דרך Source Control ב-Cursor בלי שורת פקודה; הודעות commit ברורות.

## Examples (דוגמאות קונקרטיות)

### הודעות commit טובות למתחילים

- `add first practice note`
- `update page title text`
- `save working version before agent changes`

### הודעות commit חלשות שכדאי להימנע מהן

- `fix`
- `stuff`
- `changes`

### מתי לעשות commit

- לפני שינוי גדול של Agent.
- אחרי נקודת יציבות קטנה שעובדת.
- לפני שמנסים כיוון ניסיוני שלא בטוחים לגביו.

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** כל התרגול כאן מתבצע דרך הממשק הגרפי של Source Control.

### משימה 1 — לאתחל מאגר מקומי (אחרי S03)

- **עצירה:** אחרי `Initialize Repository`.
- **מה לעשות:** לאתחל Git בתיקיית התרגול דרך הממשק.
- **איך לוודא הצלחה:** Source Control כבר לא מציע רק את כפתור האתחול, והפרויקט מזוהה כמאגר מקומי.

### משימה 2 — commit ראשון אמיתי (אחרי S04)

- **עצירה:** אחרי שלב ה-stage וה-commit.
- **מה לעשות:** לשנות קובץ קטן, להוסיף ל-stage, ולבצע commit עם הודעה ברורה של 3-6 מילים.
- **איך לוודא הצלחה:** יש נקודת שמירה חדשה בהיסטוריה, וההודעה לא גנרית כמו `fix`.

### משימה 3 — כלל "שמירה לפני ניסוי" (אחרי S06)

- **עצירה:** אחרי ההסבר על commit לפני עבודת Agent מסוכנת.
- **מה לעשות:** לכתוב לעצמכם כלל עבודה אחד שמתחיל ב-"לפני שאני מבקש שינוי גדול, אני..."
- **איך לוודא הצלחה:** הכלל מזכיר commit או save point לפני ניסוי.

### S01 | 00:00–00:40 | Hook — למה Git

- **מה על המסך:** פאנל Claude Code או רשימת קבצים עם **מספר שינויים**; או אנימציה מהירה של קבצים שנוספים/נמחקים.
- **פעולות:** גלילה על רשימת קבצים ב-diff (בלי לקרוא כל שורה).
- **קריינות:** "כשעובדים עם סוכן על הפרויקט הוא יכול לשנות הרבה קבצים בבת אחת. **Git** נותן נקודות חזרה — כמו שמירה במשחק, רק לקוד."
- **אם לא מספיק:** שקופית עם אייקון "Undo" וכיתוב Git.

### S02 | 00:40–01:35 | אנלוגיה — save במשחק

- **מה על המסך:** שתי תמונות או אייקונים: ג'ויסטיק + ענן "גרסה"; או צילום ממשק משחק "Save Game".
- **פעולות:** מעבר חלק (crossfade) בין תמונות.
- **קריינות:** "חושבים על **commit** כמו 'שמירת משחק': לפני בוס קשה שומרים נקודה. כך אם משהו השתבש, אפשר לחזור אחורה."
- **אם לא מספיק:** רק קריינות על רקע אחיד.

### S03 | 01:35–02:45 | Initialize Repository

- **מה על המסך:** סרגל צד **Source Control** (אייקון ענפים); כפתור **Initialize Repository**; אחרי לחיצה — תיקייה `.git` (אופציונלי: הצגה ב-Explorer עם "הצגת קבצים נסתרים" רק אם מתאים לקהל).
- **פעולות:** לחיצה אחת; המתנה עד שהכפתור משתנה לרשימת שינויים או שהמאגר מזוהה.
- **קריינות:** "ב-Cursor נכנסים ל-**Source Control** ולוחצים **Initialize Repository**. נוצרת תיקייה נסתרת `.git` — זה המאגר המקומי שלכם."
- **עצירת כיתה — משימה 1.**
- **אם לא מספיק:** המאגר כבר מאותחל — הציגו פרויקט חדש ריק; אם הכפתור לא מופיע — `Git: Initialize Repository` מ-Command Palette.

### S04 | 02:45–04:00 | Stage ו-Commit

- **מה על המסך:** קובץ `readme.txt` או `notes.md` עם שורה חדשה; ב-Source Control: קובץ תחת **Changes**; לחיצה על **+** (Stage); שדה הודעה; **Commit**; מעבר ל-**Staged** ריק ו-history עם commit אחד.
- **פעולות:** הקלדת הודעה ברורה (`add first practice note`); עצירה 2 שניות על ה-commit בהיסטוריה.
- **קריינות:** "קובץ משתנה מופיע תחת **Changes**. לוחצים **+** ל-stage, כותבים הודעת commit ברורה, ולוחצים **Commit**. כך נשמרת גרסה בהיסטוריה."
- **עצירת כיתה — משימה 2.**
- **אם לא מספיק:** Commit נכשל בשידור — הראו הודעת שגיאה טיפוסית והסבירו "חסר user.name" ופתרון בקצרה, או עברו להקלטה מוקדמת של commit מוצלח.

### S05 | 04:00–05:00 | צבעים + אישור ידני מול auto-accept

- **מה על המסך:** רשימת קבצים עם נקודות צבע (U/M) או אייקונים; שקופית משנייה עם "manual approve vs auto-accept" בעברית.
- **פעולות:** הצבעה עם העכבר על כל סטטוס בלי לשנות קבצים.
- **קריינות:** "הקבצים מסומנים **U** או **M** — חדש או שונה. עם **Claude Code** חשוב לא לאשר שינויים אוטומטית בלי לעבור על **diff**; בעורך הקורס נשארים על אישור ידני."

### S06 | 05:00–06:00 | שמירה לפני ניסוי

- **מה על המסך:** צ'קליסט: ☑ commit לפני שינוי גדול; תאריך/שעה דמה.
- **קריינות:** "לפני שמבקשים מהסוכן שינוי גדול — **commit**. זו נקודת בטחון לפני ניסוי."
- **עצירת כיתה — משימה 3.**

### S07 | 06:00–07:00 | מה דולגים בכוונה

- **מה על המסך:** מילים מחוקות או אפורות: branch, merge, worktree, conflict.
- **קריינות:** "**Branch**, **merge**, **worktrees**, קונפליקטים — נשארים למסלול המתקדם. כאן רק מאגר מקומי ו-commit."

### S08 | 07:00–08:00 | GitHub Desktop (אופציונלי)

- **מה על המסך:** לוגו GitHub Desktop או חלון האפליקציה עם commit פשוט (בלי repo אמיתי רגיש).
- **קריינות:** "מי שמעדיף ממשק נפרד יכול להשתמש ב-**GitHub Desktop**; העיקרון זהה — stage, commit, היסטוריה."
- **אם לא מספיק:** דלגו על הסצנה או קיצרו ל-30 שניות — רק קריינות.

### S09 | 08:00–08:30 | סיום ו-B05

- **מה על המסך:** כותרת B05.
- **קריינות:** "בשיעור הבא: חשבון, מודלים, ומה עולה כסף — בלי הפתעות בחשבונית."
