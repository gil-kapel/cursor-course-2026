# Script Master: B11 MCP Taste — One Server, Least Privilege

## Metadata

- Lesson ID: B11
- Title: טעימה: MCP אחד ובטיחות הרשאות
- Target duration: 08:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

להסביר מהו MCP; להפעיל שרת MCP אחד רלוונטי; לתעד מה השרת רשאי לקרות או לבצע; עקרון least privilege.

## Examples (דוגמאות קונקרטיות)

### משימות בטוחות יחסית לשרת MCP אחד

- "בדוק URL אחד שאני נותן."
- "הבא סיכום ממקור אחד שאישרתי מראש."
- "קרא מסמך אחד והחזר שלוש נקודות."

### שאלות שחייבים לשאול לפני הפעלה

- מה השרת יכול לקרוא?
- מה השרת יכול לכתוב או לבצע?
- האם הוא דורש סוד או token?

### מה לא לעשות בשיעור בסיסי

- להפעיל כמה שרתים יחד
- לחבר שרת שלא מבינים את ההרשאות שלו
- להדביק secrets בצ'אט או ב-commit

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** המטרה כאן היא שיקול דעת, לא כמות כלים.

### משימה 1 — להבין הרשאות לפני הפעלה (אחרי S02)

- **עצירה:** אחרי ההסבר על מה שרת יכול לראות או לעשות.
- **מה לעשות:** לכתוב בשלוש שורות: שם השרת, מה הוא יכול לקרוא, ומה הוא יכול לבצע או לגעת בו.
- **איך לוודא הצלחה:** אתם יודעים לענות על "למה דווקא השרת הזה" לפני שלחצתם enable.

### משימה 2 — משימה אחת, מוגדרת היטב (אחרי S04)

- **עצירה:** אחרי הדוגמה של משימה אחת עם MCP.
- **מה לעשות:** להפעיל את השרת ולבצע משימה אחת קטנה ומוגדרת.
- **איך לוודא הצלחה:** התקבלה תוצאה אחת ברורה, בלי להפוך את זה לחיפוש פתוח או שרשרת כלים.

### משימה 3 — כיבוי או תיעוד (אחרי S06)

- **עצירה:** אחרי ההסבר מתי לכבות.
- **מה לעשות:** לבחור אחד: או לכבות את השרת בסיום, או לכתוב למה משאירים אותו פעיל.
- **איך לוודא הצלחה:** יש החלטה מודעת, לא "השארתי כי שכחתי".

### S01 | 00:00–00:40 | Hook

- **מה על המסך:** תרשים "שקע אחד" / USB-C; לוגו Cursor + Claude Code קטן.
- **פעולות:** מעבר שקופיות או זום על תרשים; בלי פתיחת טרמינל.
- **קריינות:** "MCP זה פרוטוקול שמחבר את **Claude Code** לכלים מחוץ לפרויקט: דפדפן, מסמכים, חיפוש. **חשוב:** לפי התיעוד הרשמי של ההרחבה, **הגדרת שרתי MCP מלאה עדיין דרך ה-CLI** — וההרחבה משתמשת בהגדרות שנשמרו שם. זה עוצמתי — ולכן נלמד עם **שרת אחד** ובהרשאות מינימליות."
- **אם לא מספיק:** קצרו את רשימת הדוגמאות ("דפדפן, מסמכים…") לשתיים בלבד.

### S02 | 00:40–01:50 | What a Server Can See or Do

- **מה על המסך:** טבלה או רשימה: "קריאה" / "כתיבה" / "הרצה" עם דוגמה אחת לכל עמודה.
- **פעולות:** גלילה על הרשימה; אופציונלי פאנל Ask עם שאלה "מה השרת הזה רשאי?"
- **קריינות:** "לפני שמפעילים — קוראים בתיעוד או שואלים ב-Ask: מה השרת יכול לקרוא, לכתוב, להריץ. אם לא ברור, לא מדליקים."
- **עצירת כיתה:** **משימה 1** — להבין הרשאות לפני הפעלה.
- **אם לא מספיק:** שקופית סטטית + קריאת שלוש השאלות מהדוגמאות בראש המסמך; בלי Ask חי.

### S03 | 01:50–03:00 | Configure ONE Server via CLI, Then Use Extension

- **מה על המסך:** מסוף משולב — פקודות `claude` / עריכת קובץ הגדרות MCP לפי [המדריך הרשמי](https://code.claude.com/docs/en/vs-code); מעבר ל־VS Code עם פאנל Claude Code (אין להציג מפתחות).
- **פעולות:** גלילה על קובץ config או פלט CLI; סגירת קובץ עם סודות לפני שיתוף מסך.
- **קריינות:** "בשלב הבסיסי: מגדירים **שרת MCP אחד דרך ה-CLI** של Claude Code — לפי התיעוד, כך ההרחבה תזהה את השרת. לא מפעילים חמישה בבת אחת. אחרי ההגדרה, חוזרים ל-**פאנל Claude Code** ומריצים משימה אחת שמשתמשת בכלי — בלי להדביק secrets בצ'אט."
- **אם לא מספיק:** הקלטה מוקדמת בלבד; בכיתה — הראו רק את עמוד התיעוד ואת הנתיב לקובץ ההגדרות בלי עריכה חיה.

### S04 | 03:00–04:05 | One Concrete Task

- **מה על המסך:** פאנל Claude Code עם פרומפט אחד ממוקד (URL / מסמך מאושר); אינדיקציה שהכלי הופעל (אייקון/טקסט תשובה).
- **פעולות:** שליחת הודעה אחת; גלילה בתוצאה; עצירה אחרי תשובה אחת מלאה.
- **קריינות:** "משימה אחת מוגדרת: למשל 'הבא סיכום ממקור אחד שאני מאשר' או 'בדוק URL שאני נותן'. לא סריקה אינסופית של האינטרנט."
- **עצירת כיתה:** **משימה 2** — משימה אחת, מוגדרת היטב.
- **אם לא מספיק:** תארו את המשימה בקול והראו צילום מסך של תוצאה; בלי חיבור חי לאינטרנט.

### S05 | 04:05–05:00 | Secrets and Tokens

- **מה על המסך:** שקופית: "לא בצ'אט / לא ב-Git" + אייקון מנעול; או ממשק env vars מטושטש.
- **פעולות:** ללא הקלדת מפתחות; הצבעה על מקום מותר להגדרה.
- **קריינות:** "אם השרת דורש API key — לא מדביקים בצ'אט ציבורי ולא שומרים ב-commit. משתמשים בהגדרות סביבה או שדות מאובטחים לפי המוצר."
- **אם לא מספיק:** משפט אחד + מעבר ל-S06.

### S06 | 05:00–06:00 | When to Turn Off

- **מה על המסך:** הגדרות MCP / רשימת שרתים עם מיתוג כבוי; או אנימציה "שרת כבוי בסוף היום".
- **פעולות:** כיבוי מודגם (או הסבר על הכפתור בלי לשבור דמו של כיתה).
- **קריינות:** "מכבים שרתים שלא בשימוש. כל שרת הוא משטח התקפה פוטנציאלי אם מישהו מריץ פרומפט זדוני."
- **עצירת כיתה:** **משימה 3** — כיבוי או תיעוד.
- **אם לא מספיק:** רק קריינות + שאלה לכיתה "מי יכבה בסוף?" בלי שיתוף מסך.

### S07 | 06:00–06:55 | חפיפה למתקדם (A06 / A10)

- **מה על המסך:** כרטיס "מסלול מתקדם" + מילים: מטריצת MCP, תוספים, שומרי איכות.
- **פעולות:** דיבור מול שקופית.
- **קריינות:** "במסלול המתקדם תבנו מטריצת MCP, תוספים, ושומרי איכות. כאן רק טעימה ושפה נכונה."
- **אם לא מספיק:** 30 שניות; דלגו על רשימת המילים.

### S08 | 06:55–07:45 | Chrome DevTools MCP Mention Only

- **מה על המסך:** אייקון דפדפן + מילה "מאוחר יותר"; בלי התקנה.
- **פעולות:** ללא דמו.
- **קריינות:** "קיימים שרתים שמחוברים לדפדפן לדיבוג עמוק — נגיע אליהם אחרי שיש ביטחון בבסיס."
- **אם לא מספיק:** דלגו לחלוטין אם חורגים מזמן; זה תוספת מוטיבציה בלבד.

### S09 | 07:45–08:30 | סיום ו-B12

- **קריינות:** "בשיעור הבא נעלה את הקוד ל-GitHub — צעד אחר צעד. הפריסה לכתובת חיה תהיה ב-B13 עם Vercel."
