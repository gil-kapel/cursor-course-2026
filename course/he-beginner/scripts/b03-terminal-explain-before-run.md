# Script Master: B03 Terminal Without Fear — Explain Before Run

## Metadata

- Lesson ID: B03
- Title: המסוף בלי פחד: "הסבר לפני הרצה"
- Target duration: 08:00
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

להפחית Terminal Anxiety: לפני הרצת פקודה לא מוכרת — לשאול ב-Ask מה היא עושה, האם היא הרסנית, ומה לצפות.

## Examples (דוגמאות קונקרטיות)

### פקודות יחסית בטוחות לשיעור הדגמה

- `ls` או `dir` כדי לראות קבצים.
- `pwd` כדי לראות איפה אתם נמצאים.
- `echo hello` כדי לראות פלט פשוט.

### דוגמאות לשאלות נכונות לפני הרצה

- "הסבר מה כל חלק בפקודה הזאת עושה."
- "האם הפקודה הזאת יכולה למחוק או לשנות קבצים?"
- "איזה פלט סביר אני אמור לראות אם הכול תקין?"

### דגלים אדומים שכדאי לעצור עליהם

- `sudo`
- נתיבים שנראים מחוץ לתיקיית הפרויקט
- הורדה מאתר לא מוכר
- פקודות מחיקה כמו `rm -rf`

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** כל משימה כאן היא קצרה ובטוחה. לא להכניס פקודות הרסניות לצורך הדגמה.

### משימה 1 — לשאול לפני Enter (אחרי S03)

- **עצירה:** אחרי הדגמת שאלה בפאנל Claude Code עם פקודה מודבקת.
- **מה לעשות:** להדביק פקודה אחת פשוטה ל-Ask ולבקש הסבר, סיכון ופלט צפוי.
- **איך לוודא הצלחה:** קיבלתם תשובה שמסבירה מה הפקודה עושה לפני שהרצתם אותה.

### משימה 2 — להריץ פקודת מידע בלבד (אחרי S04)

- **עצירה:** אחרי הדוגמה של `ls` / `dir` / `pwd`.
- **מה לעשות:** להריץ פקודת מידע אחת בלבד מתוך תיקיית הפרויקט.
- **איך לוודא הצלחה:** אתם רואים פלט סביר כמו נתיב או רשימת קבצים, בלי שינוי בקבצים עצמם.

### משימה 3 — לזהות דגל אדום או שורת סיום (אחרי S05-S06)

- **עצירה:** אחרי הדגלים האדומים וההסבר על קריאת השורות האחרונות.
- **מה לעשות:** לבחור אחד: או לכתוב דגל אדום אחד שיגרום לכם לעצור, או לצלם/לציין את שתי השורות האחרונות של הפלט שקיבלתם.
- **איך לוודא הצלחה:** יש לכם הרגל בדיקה ברור: לעצור על סיכון, או לקרוא את סוף הפלט לפני שממשיכים.

### S01 | 00:00–00:35 | Hook — המסוף לא "שובר הכול"

- **מה על המסך:** מסוף משולב ב-Cursor, ריק או עם שורת `>` / `$` מהבהבת; רקע כהה או בהיר לפי ערכת הנושא.
- **פעולות:** סמן מהבהב בלבד — בלי להקליד.
- **קריינות:** "המסוף נראה כמו מקום שבו אפשר 'לשבור את המחשב'. בפועל, ברוב המקרים פקודות הדרכה לא הורסות כלום — אבל כדאי לדעת מה רצים. נלמד הרגל אחד שמשנה הכול."
- **אם לא מספיק:** תמונה סטטית של מסוף + קריינות.

### S02 | 00:35–01:30 | למה המסוף קיים + CLI ↔ IDE

- **מה על המסך:** פיצול: למעלה/משמאל — פאנל Claude Code עם טקסט שמציע פקודה; למטה/מימין — אותה פקודה מודבקת במסוף או פלט אחרי הרצה **בטוחה** שכבר הוכן.
- **פעולות:** הדגשה על הקשר בין הצעת הסוכן לבין שורת המסוף; אופציונלי: הקלדת `claude` בלי אימות — רק Enter לעצירה מיידית או בלי הרצה.
- **קריינות:** "המסוף נותן להריץ כלים ולראות פלט באותו חלון. **Claude Code** מציע לפעמים פקודות שמתבצעות שם. מי שעובד גם מחוץ לעורך לפעמים משתמש ב־`cursor` ב־PATH כדי לפתוח את הפרויקט — כאן רק מחברים את הנקודות, בלי צלילה ל-legacy CLI."
- **אם לא מספיק:** שקופית עם שני חצים: "הצעה בפאנל" → "מסוף".

### S03 | 01:30–02:40 | שאלה בפאנל לפני Enter

- **מה על המסך:** פאנל Claude Code פתוח; מצב **שאלה/צ'אט** בלי מצב "הרץ הכול"; טקסט prompt מוכן עם פקודה מודבקת (למשל `npm install` דמה — עדיף להחליף ב-`ls` אם חוששים).
- **פעולות:** הדבקה מסודרת; גלילה על רשימת הבקשות ב-prompt (הסבר, סיכון, פלט).
- **קריינות:** "לפני Enter — שואלים בפאנל: מה כל חלק בפקודה עושה, מה הסיכון, ומה הפלט הצפוי. לא מריצים עד שיש תשובה שאתם מבינים."
- **עצירת כיתה — משימה 1.**
- **אם לא מספיק:** אין חיבור ל-API — הציגו את ה-prompt על שקופית והסבירו את התשובה הצפויה בעל-פה.

### S04 | 02:40–03:45 | פקודת מידע בטוחה

- **מה על המסך:** Mac/Linux: `ls` או `pwd`; Windows: `dir` או `cd` בלי ארגומנטים מסוכנים. הפלט: רשימת קבצים או נתיב.
- **פעולות:** הקלדה איטית או הדבקה; Enter; המתנה לפלט.
- **קריינות:** "עכשיו פקודת מידע בלבד: `ls`, `dir`, או `pwd` — רואים רשימת קבצים או נתיב, בלי לשנות קבצים. זה הבסיס לביטחון במסוף."
- **עצירת כיתה — משימה 2.**
- **אם לא מספיק:** המסוף לא בתיקיית הפרויקט — הריצו `cd` לנתיב הפרויקט לפני הדגמה (הכינו את הנתיב מראש).

## Prompt Asset

`Here is a terminal command I'm about to run inside my project folder on [macOS/Windows]: [PASTE COMMAND]. Explain in simple Hebrew: (1) what each part does, (2) whether it can delete or change files outside this folder, (3) expected output if it works, (4) whether I should run it now or ask for a safer alternative.`
