# קורס Cursor מתקדם (עברית)

## מיקום במסלול הלמידה

זהו **מסלול המתקדם** של חבילת הקורס: לא "איך להשתמש ב-Cursor", אלא איך **לספק עבודה מקצועית בעזרת Cursor** בתוך פרויקט אמיתי, עם אילוצים, סיכון, בדיקות, review, handoff ו-release.

**מסלול בסיסי (מומלץ לפני או במקביל ללא רקע טכנולוגי):** [`../../he-beginner/docs/course-outline.md`](../../he-beginner/docs/course-outline.md)  
**חפיפה מכוונת בין המסלולים:** [`../../he-beginner/docs/cross-track-overlap.md`](../../he-beginner/docs/cross-track-overlap.md)

## קדם-דרישות מומלצות

- היכרות בסיסית עם Cursor: פתיחת פרויקט, שימוש ב-Ask/Plan/Agent/Debug, וקריאת diff.
- Git ברמת שימוש יומיומית בסיסית: commit, הבנת שינויים, ונכונות לעבוד עם branch.
- עדיף השלמת **המסלול הבסיסי** לפחות עד שיעור `B08`, או ניסיון מקביל שמכסה את אותם יסודות. לפני שיעור הפריסה המתקדם (13) מומלץ גם `B12–B13` (GitHub + Vercel) כדי שיהיה קו בסיס לפריסה.

## מטרת הקורס

להביא משתמש שכבר מרגיש בטוח ב-Cursor לעבודה עצמאית על **פיצ'ר בתוך codebase קיים**: להבין את המערכת, לפרק את העבודה לפרוסות בטוחות, לממש עם guardrails, לאמת, להתאושש מתקלות, להכין handoff, ולשחרר גרסה בביטחון.

## הבטחת הערך של המסלול

במסלול הבסיסי ההבטחה היא: **"אני יודע לעבוד עם Cursor בלי פחד."**  
במסלול המתקדם ההבטחה היא: **"אני יודע לספק שינוי מקצועי עם Cursor."**

כלומר, התוצר כאן איננו רק prompt טוב, אלא:

- plan שניתן לבצע
- diff שאפשר לסקור
- ולידציה שאפשר לסמוך עליה
- handoff ברור לאדם אחר
- release עם בדיקה ודרך חזרה

## קהל יעד

- מפתחים ואנשי מוצר שכבר עובדים עם Cursor ורוצים **workflow של delivery**, לא רק שימוש נקודתי.
- צוותים קטנים שרוצים דרך עקבית לעבוד עם rules, memory, MCPs, review ו-handoffs.
- יוצרי תוכן ומדריכים שרוצים להבין איך להפוך workflow טוב ל-pipeline שניתן לשכפל גם לפרסום וללוקליזציה.

## מה לא נלמד כאן לעומק

- סיור ממשק איטי ב-Cursor או הסברים ראשוניים על כל כפתור.
- Git כ-save points בלבד דרך GUI.
- פרויקטים "צעצוע" כתחליף לעבודה על repo אמיתי.

הנושאים האלה שייכים בעיקר למסלול הבסיסי. כאן מניחים שהם כבר ברקע, או מכסים אותם ב-bridge קצר בלבד.

## מבנה הקורס

- פורמט: 14 סרטונים
- אורך מומלץ: 5 עד 10 דקות לסרטון
- סגנון: מצב אמת קודם, אחר כך פירוק החלטות, ואז checklist לביצוע
- חוט מקשר: רוב הקורס עובד על **repo מסכם אחד** שמלווה את הלומד משיעור 01 עד שיעור 13

## סטנדרט שיעור

- לכל שיעור יש `Script Master`, `Prompt Asset`, `Recording Notes`, ו-`Action Item`.
- ה-`Scene List` משמש גם כבסיס לחלוקת קריינות ל-TTS.
- הנחיות הפקה וקריינות משותפות נמצאות ב-`course/he-advanced/docs/recording-and-elevenlabs-guide.md`.
- עקביות מונחים נשמרת לפי `course/he-advanced/docs/hebrew-tts-glossary.md`.

## עקרון פדגוגי

המסלול המתקדם אינו "אותו חומר מהר יותר", אלא שינוי ברמת האתגר:

- יותר codebase קיים, פחות greenfield
- יותר אילוצים וסיכון, פחות הדגמות סטריליות
- יותר ולידציה ו-review, פחות "תסמכו על ה-agent"
- יותר handoff ו-release, פחות עבודה אישית מבודדת

## סדר השיעורים

### 01. פתיחת הקורס דרך capstone אמיתי

- מטרה: לפתוח מייד ב-repo אמיתי, feature request מוגדר, וסטנדרט delivery ברור.
- תוצאה לומדת: הסטודנט מבין מהו ה-capstone שמלווה את הקורס ומהי רמת האיכות המצופה בסיום.
- דמו מרכזי: קריאת ticket, מעבר על repo קיים, והצגת היעד הסופי: feature, review, deploy ו-handoff.

### 02. Bridge מהיר: sanity על setup, modes ו-context

- מטרה: לסגור פערים למי שלא הגיע מהמסלול הבסיסי בלי להפוך את הקורס לעוד קורס היכרות.
- תוצאה לומדת: הסטודנט יודע לבדוק במהירות ש-Cursor, המנוי, ה-mode selection והקשר הפרויקט מוכנים לעבודה.
- דמו מרכזי: checklist קצר של readiness במקום סיור מלא בממשק.
- הערה: לבוגרי המסלול הבסיסי זהו שיעור רפרנס/ריענון, לא ליבת הערך של הקורס.

### 03. קריאת codebase לא מוכר עם Ask ו-Plan

- מטרה: ללמד איך נכנסים לפרויקט קיים בלי לקפוץ ישר ל-Agent.
- תוצאה לומדת: הסטודנט יודע לבנות map מהיר של מודולים, נקודות כניסה, וסיכונים לפני שינוי.
- דמו מרכזי: חקירת repo קיים, זיהוי שכבות, ושאלות הבהרה לפני מימוש.

### 04. פירוק feature לפרוסות בטוחות

- מטרה: להפוך בקשה רחבה לתוכנית delivery שאפשר לבצע בלי לשבור חצי מערכת.
- תוצאה לומדת: הסטודנט יודע לנסח acceptance criteria, לזהות dependency order, ולבחור slice ראשון.
- דמו מרכזי: מ-ticket עמום אל plan מסודר עם שלבי מימוש קטנים.

### 05. אסטרטגיית context ומודלים ל-repo אמיתי

- מטרה: ללמד בחירה מודעת של context, model ו-scope עבור משימות גדולות יותר.
- תוצאה לומדת: הסטודנט יודע מתי להישאר על `Auto`, מתי לבחור מודל מפורש, ואיך לקצץ רעש כדי לשפר דיוק ולהוריד עלות.
- דמו מרכזי: אותה משימה עם context רחב מדי מול handoff ממוקד ו-routing מודלים לפי שלב העבודה.

### 06. מימוש slice ראשון עם guardrails

- מטרה: להראות איך עוברים מ-plan לביצוע מדורג עם עצירות בטיחות.
- תוצאה לומדת: הסטודנט יודע להנחות Agent לעבוד בשכבה אחת או שתיים בלבד, לעצור אחרי slice, ולתעד מה השתנה.
- דמו מרכזי: מימוש חלק ראשון של הפיצ'ר עם stop conditions, diff review ו-checkpoint.

### 07. ולידציה: tests, lint ו-checklists לפני שממשיכים

- מטרה: לבנות הרגל של validation מיידי ולא "נבדוק בסוף".
- תוצאה לומדת: הסטודנט יודע להתאים בדיקות לשינוי, לבקש מ-Cursor לאמת את עצמו, ולהבדיל בין smoke check לבין confidence אמיתי.
- דמו מרכזי: הרצת lint, test או בדיקה ידנית ממוקדת אחרי slice אחד.

### 08. דיבוג רגרסיות והתאוששות מבלאגן

- מטרה: ללמד חזרה לשליטה כששינוי AI יצר נזק, בלבול או התפזרות.
- תוצאה לומדת: הסטודנט יודע לצמצם failure, להעלות היפותזות, להגדיר תיקון קטן, ולחזור ללופ בלי להחריף את הבעיה.
- דמו מרכזי: רגרסיה אמיתית ב-capstone, root cause, ותיקון בטוח.

### 09. Git, branches ו-worktrees לעבודה מקבילה

- מטרה: להראות איך שומרים על main נקי תוך כדי ניסויים, bugfixes ו-parallel attempts.
- תוצאה לומדת: הסטודנט יודע מתי לפתוח branch רגיל, מתי worktree, ואיך לנהל כמה מהלכים בלי בלגן.
- דמו מרכזי: worktree אחד לפיצ'ר, worktree שני לבאג דחוף, והשוואה ביניהם.

### 10. Code Review ושומרי סף לפני merge

- מטרה: להכניס review כשלב הכרחי ולא כתוספת נחמדה.
- תוצאה לומדת: הסטודנט יודע לבקש review ממוקד סיכונים, לזהות רגרסיות, ולדרוש הוכחת תקינות לפני merge.
- דמו מרכזי: diff שנראה תקין במבט ראשון אבל מסתיר בעיית behavior או תחזוקה.

### 11. הקשר פרויקט מתקדם: rules, memory, MCPs ותוספים כחוזה עבודה

- מטרה: לשדרג project context מ"נוחות" ל"חוזה תפעולי" לצוות ול-repo.
- תוצאה לומדת: הסטודנט יודע להגדיר rules שמונעים קפיצות בין שכבות, memory שמתעד החלטות יציבות, ו-MCPs עם least privilege.
- דמו מרכזי: עדכון context לפרויקט אחרי שהבנו את הכאבים האמיתיים של ה-capstone.

### 12. עבודה בצוות: handoff, PR prep ותקשורת סביב שינוי

- מטרה: ללמד איך מעבירים משימה, לא רק איך פותרים אותה לבד.
- תוצאה לומדת: הסטודנט יודע להכין handoff, לנסח PR summary, ולשמור על הקשר שעובר בין אנשים ולא נתקע בתוך thread אחד.
- דמו מרכזי: הכנת handoff package לאדם אחר או ליום עבודה הבא.

### 13. פריסה, אימות ורולבק — AWS בסיסי (S3 + CloudFront)

- מטרה: לחבר את ה-feature שנבנה לשלב release עם **תשתית מוגבלת ב-AWS**: אתר סטטי ב-**S3** מאחורי **CloudFront**, אימות אחרי deploy, ומנטליות **רולבק** — בלי לכסות את כל הענן.
- תוצאה לומדת: הסטודנט מבין Region, אחסון מול CDN, סודות מחוץ ל-repo, smoke checks אחרי פריסה, ומתי לא לדחוף ל-production.
- דמו מרכזי: נתיב אחד מצומצם S3 → CloudFront → URL חי, בדיקה קצרה, ודיון קצר על החזרת גרסה יציבה.
- הקשר: במסלול הבסיסי נלמדה פריסה מהירה עם **Vercel** (B13); כאן שכבת ענן "אמיתית" יותר לצורך שליטה וסיכון.
- סקריפט: [`../scripts/13-aws-deployment-s3-cloudfront.md`](../scripts/13-aws-deployment-s3-cloudfront.md).

### 14. הרחבה: publishing, export ולוקליזציה של workflow

- מטרה: להראות איך אורזים workflow מוצלח לפורמטים נוספים, שפות נוספות, או שימוש חוזר בצוות.
- תוצאה לומדת: הסטודנט מבין איך להפריד תוצרי מקור, captions, glossary ו-assets כך שאפשר יהיה לשכפל את התהליך ולא לבנות הכול מחדש.
- דמו מרכזי: מעבר מ-workflow עברי קיים ל-pipeline מסודר של export ולוקליזציה.

### Arc 2: Controlled Execution

- שיעורים: 06 עד 10

- תוצר: project context כסטנדרט צוותי, handoff ברור, deploy עם rollback, ויכולת לשכפל workflow הלאה

## חפיפה עם המסלול הבסיסי

- החפיפה הישירה הצטמצמה בכוונה. במקום לפתוח שוב בהתקנה, מנוי, modes ו-context כהקדמה ארוכה, המסלול המתקדם דוחס אותם ל-**Bridge מהיר בשיעור 02**.
- הערך המרכזי של המסלול המתקדם מתחיל כבר ב-**שיעור 03**: קריאת codebase, פירוק feature, context עבור repo אמיתי, validation, recovery, worktrees, review, handoff ו-release.
- בוגרי המסלול הבסיסי יכולים לרפרף על **02** ולעבור כמעט מייד לליבה של הקורס.
- טבלת מיפוי מפורטת: [`../../he-beginner/docs/cross-track-overlap.md`](../../he-beginner/docs/cross-track-overlap.md).

## תוצרים צפויים מהלומד

- להיכנס ל-codebase לא מוכר ולהבין אותו בלי לקפוץ ישר לשינוי.
- לפרק feature לפרוסות בטוחות עם acceptance criteria ברורים.
- לבחור mode, model ו-context לפי סוג המשימה והסיכון.
- לממש שינוי קטן עם guardrails ולעצור בזמן.
- לאמת כל שלב דרך tests, lint, smoke checks או בדיקה ידנית ממוקדת.
- להתאושש מרגרסיה בלי להעמיק את הבלגן.
- לעבוד עם branches ו-worktrees בבטחה.
- להכין review, handoff ו-deploy כמו איש צוות מקצועי.
- לפרוס אתר סטטי בנתיב מוגבל ב-AWS (S3 + CloudFront) עם אימות ורעיון רולבק.
