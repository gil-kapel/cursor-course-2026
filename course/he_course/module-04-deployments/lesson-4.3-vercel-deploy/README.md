## Syllabus topic

Turn the prepared repo into a live app with **Vercel**: connect the GitHub project, confirm the build settings, add environment variables in the right place, deploy, and verify the real public URL instead of assuming success.

## Practical outcomes

- Connect a GitHub repo to Vercel without guessing build settings.
- Move runtime secrets to Vercel environment variables.
- Understand what `build command` and `output directory` actually mean in practice.
- Make one deployment and verify the live app with a real checklist.
- Learn the fastest debugging loop when deploy succeeds but behavior breaks.
