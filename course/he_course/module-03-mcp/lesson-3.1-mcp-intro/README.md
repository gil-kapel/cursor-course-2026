## Syllabus topic

What MCP actually is, why it matters inside Cursor, how to add and verify a server, and how to integrate any MCP into a real project workflow instead of treating it like a one-off demo.

## Practical outcomes

- Understand MCP as the bridge between Cursor and external tools or live data sources.
- Add one MCP server in Cursor and verify that it is available before using it in a task.
- Learn a repeatable integration flow: choose a workflow problem, pick the right MCP, connect it, test it, and turn its output into a project artifact.
- Understand the difference between using an MCP in chat and integrating it into the project's real delivery loop.
- Build a checklist the student can reuse for other MCPs later, such as GitHub, docs, browser, database, design, or product tools.

## Student task

- Choose one simple workflow problem in your project.
- Pick one MCP that fits that problem.
- Connect it in Cursor and verify that it works with one tiny request.
- Save one useful output from that MCP, such as notes, a task list, a UI breakdown, or a short decision.
- Write one sentence explaining where that output fits in your project workflow.

## Prompt

`You are an MCP setup assistant. Goal: help me choose and test one MCP for one real workflow problem in my project. Context: my problem is [problem]. Output: the best MCP choice, one tiny test request, and one example of a useful output I should save back into the project. Constraints: keep it beginner-friendly, use one MCP only, and focus on one clear task.`
