# חפיפה בין מסלול בסיסי למסלול מתקדם

מסמך זה מגדיר **חפיפה מכוונת אך מוגבלת**: יש יסודות משותפים לשני המסלולים, אבל המסלול המתקדם נבנה מחדש סביב **delivery מקצועי בתוך repo אמיתי**, ולא סביב לימוד Cursor מחדש.

## עקרון

| במסלול הבסיסי | במסלול המתקדם |
| -------------- | -------------- |
| קצב איטי, מטאפורות, בניית ביטחון | קצב מקצועי, אילוצים, איכות ושחרור |
| Greenfield קטן ומבוקר | Codebase קיים, feature אמיתי, סיכון אמיתי |
| מינימום הכרחי כדי לעבוד בלי פחד | סטנדרט עבודה שאפשר לסמוך עליו בצוות |

## מה חופף באמת

החפיפה הישירה היא בעיקר ביסודות:

- מה Cursor יודע לעשות ומה לא
- בחירת mode ומודל
- הגדרת context בסיסי
- בטיחות בסיסית עם Git ו-MCP

אבל במסלול המתקדם היסודות האלה נדחסים ל-**Bridge מהיר** ולא תופסים שוב את מרכז הבמה.

## המלצה למדריך

- מסלול **בסיסי → מתקדם**: להתחיל ב-`01`, לרפרף על `02`, ואז להיכנס ישר ללב הקורס ב-`03`.
- מסלול **רק מתקדם** למפתחים ותיקים: אפשר להשתמש ב-`02` כ-checklist קצרה ואז להמשיך ל-`03`.
- אם לומד כבר מרגיש חזק ב-Cursor אבל חלש ב-delivery, לא לדלג על `04–13` גם אם היסודות מוכרים לו.
