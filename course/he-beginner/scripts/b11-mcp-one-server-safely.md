# Script Master: B11 MCP Taste — One Server, Least Privilege

## Metadata

- Lesson ID: B11
- Title: טעימה: MCP אחד ובטיחות הרשאות
- Target duration: 08:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

להסביר מהו MCP; להפעיל שרת MCP אחד רלוונטי; לתעד מה השרת רשאי לקרות או לבצע; עקרון least privilege.

## Examples (דוגמאות קונקרטיות)

### משימות בטוחות יחסית לשרת MCP אחד

- "בדוק URL אחד שאני נותן."
- "הבא סיכום ממקור אחד שאישרתי מראש."
- "קרא מסמך אחד והחזר שלוש נקודות."

### שאלות שחייבים לשאול לפני הפעלה

- מה השרת יכול לקרוא?
- מה השרת יכול לכתוב או לבצע?
- האם הוא דורש סוד או token?

### מה לא לעשות בשיעור בסיסי

- להפעיל כמה שרתים יחד
- לחבר שרת שלא מבינים את ההרשאות שלו
- להדביק secrets בצ'אט או ב-commit

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** המטרה כאן היא שיקול דעת, לא כמות כלים.

### משימה 1 — להבין הרשאות לפני הפעלה (אחרי S02)

- **עצירה:** אחרי ההסבר על מה שרת יכול לראות או לעשות.
- **מה לעשות:** לכתוב בשלוש שורות: שם השרת, מה הוא יכול לקרוא, ומה הוא יכול לבצע או לגעת בו.
- **איך לוודא הצלחה:** אתם יודעים לענות על "למה דווקא השרת הזה" לפני שלחצתם enable.

### משימה 2 — משימה אחת, מוגדרת היטב (אחרי S04)

- **עצירה:** אחרי הדוגמה של משימה אחת עם MCP.
- **מה לעשות:** להפעיל את השרת ולבצע משימה אחת קטנה ומוגדרת.
- **איך לוודא הצלחה:** התקבלה תוצאה אחת ברורה, בלי להפוך את זה לחיפוש פתוח או שרשרת כלים.

### משימה 3 — כיבוי או תיעוד (אחרי S06)

- **עצירה:** אחרי ההסבר מתי לכבות.
- **מה לעשות:** לבחור אחד: או לכבות את השרת בסיום, או לכתוב למה משאירים אותו פעיל.
- **איך לוודא הצלחה:** יש החלטה מודעת, לא "השארתי כי שכחתי".
