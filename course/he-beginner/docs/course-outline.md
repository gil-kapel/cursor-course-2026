# קורס בסיסי: Cursor + Claude Code for VS Code (עברית)

## מיקום במסלול הלמידה

זהו **מסלול הבסיס** של חבילת הקורס. הוא מיועד ללומדים **ללא רקע טכנולוגי פורמלי** שרוצים לעבוד עם **Claude Code** (Anthropic) **בתוך Cursor**: מתקינים את ההרחבה הרשמית **Claude Code for VS Code** (תואמת VS Code) מתוך ממשק ההרחבות של Cursor. Cursor מבוסס VS Code — אותה הרחבה עובדת גם ב-VS Code נקי; במסלול הזה נבחר ב-**Cursor** כעורך הראשי.

לאחריו מומלץ להמשיך ל-[קורס Cursor מתקדם](../../he-advanced/docs/course-outline.md), שמרחיב את אותם עקרונות בעומק מקצועי — worktrees, MCP עומק, פריסה AWS, צוות (מעבר מוסבר בחפיפה).

חפיפה מכוונת בין המסלולים מתועדת ב-[`cross-track-overlap.md`](cross-track-overlap.md).

## מטרת הקורס

להביא לומד מ**אפס ניסיון בסביבת פיתוח** עד ליכולת:

- לנווט ב-**Cursor** (תיקיות, עורך, פאנל Claude Code, מסוף משולב) בביטחון.
- לראות שינויי קוד (diff) ולאשר או לדחות בצורה מבוקרת לפני שמקבלים שינוי מהסוכן.
- לנהל **גיבוי מקומי** עם Git דרך הממשק הגרפי בלבד (נקודות שמירה), בלי תסריט של ענפים ומיזוגים.
- להריץ זרימת עבודה **שאלה → תכנון → ביצוע מבוקר → דיבוג** עם **Claude Code**.
- לבנות פרויקט קטן מקומי, להעלות אותו ל-GitHub, ולפרסם גרסה ראשונה בעזרת ספק אחד (Vercel) בשיעור ייעודי.

## קהל יעד

- מנהלי מוצר, שיווק, מחקר, יזמות — ללא רקע בקוד.
- משתמשים מנוסים ב-ChatGPT/Claude בדפדפן שרוצים **הקשר פרויקט מלא** וסוכן שעורך קבצים **ב-Cursor עם Claude Code**.
- כל מי שסובל מ"חרדת מסוף" או מ"פחד לשבור משהו" וצריך **רשת ביטחון** לפני שימוש בסוכן.

## מה לא נלמד כאן (נשמר למסלול המתקדם)

- **Claude Code ב-CLI** כמסלול עבודה ראשי — מוזכר ב-B01; כאן המיקוד הוא **ההרחבה בעורך** (Cursor).
- Worktrees, ענפים מקבילים וזרימות Git מורכבות.
- מטריצת MCPs מלאה, Skills מורכבים, ותוספים רחבים — רק טעימה.
- תמחור ומנויים לעומק מעבר ל-Claude/Anthropic בקשר לשימוש ב-Claude Code (רק עקרונות).
- עבודת צוות, handoffs, pipeline פרסום/לוקליזציה של קורסים.

## מבנה הקורס

- פורמט מתוכנן: **13 סרטונים** (ניתן לפצל מודולים ארוכים לשני סרטונים בפועל).
- אורך מומלץ: **6 עד 12 דקות** לסרטון (קצב איטי יותר מהמסלול המתקדם).
- סגנון: דמו מעשי → הסבר קצר → צ'קליסט → משימת בית אחת.

## סדר השיעורים

### B01. ממצרכן צ'אט למנצח תהליך — CLI, עורך, ולמה Cursor

- מטרה: **Claude Code ב-CLI** — מה זה (מסוף, אוטומציה) ולמה למתחילים קשה יותר בלי diff ויזואלי; למה **Claude Code for VS Code** בתוך **Cursor** (או VS Code) נוח יותר לעבודת פרויקט; צ'אט בדפדפן מול סוכן על תיקייה; חלון הקשר; סטאק המסלול (**Cursor + הרחבת Anthropic**).
- תוצר: הלומד מבין שתפקידו לתאר דרישות, לאשר תוכניות ולנהל סיכון — לא "לזכור תחביר".
- דמו: השוואה קצרה — טרמינל מול פאנל diff; דפדפן מול Cursor.

### B02. Cursor + Claude Code: התקנה וסיור בממשק

- מטרה: התקנת **Cursor** (אם צריך) והרחבת **Claude Code for VS Code** מהמארקטפלייס; אנלוגיות — Explorer, עורך, פאנל Claude Code, מסוף.
- תוצר: תיקיית פרויקט ריקה ב-Cursor; Claude Code פעיל; קובץ טקסט ראשון.
- דמו: `Open Folder`, Extensions, פאנל Claude Code, מסוף משולב, שמירה.
- נכסים: [`scripts/b02-vscode-claude-code-ui-tour.md`](../scripts/b02-vscode-claude-code-ui-tour.md).
- חפיפה: מקביל לרוחב של שיעורי **מתקדם 01–02** (כאן איטי ומילולי יותר).

### B03. המסוף בלי פחד: "הסבר לפני הרצה"

- מטרה: להפחית Terminal Anxiety — לפני פקודה לא מוכרת, לשאול את **Claude** מה היא עושה ואם היא הרסנית.
- תוצר: הרגל עבודה אחד (תבנית prompt) לפני הרצה.
- דמו: פקודה פשוטה אחת (למשל רשימת קבצים) עם הסבר מהסוכן.

### B04. Git כרשת ביטחון (נקודות שמירה)

- מטרה: Git **רק** כמכונת זמן / Save points; בלי ענפים, merge, origin בעומק.
- תוצר: Commit ראשון דרך **Source Control** ב-Cursor; הודעת commit משמעותית; הבנת צבעי סטטוס קבצים.
- דמו: שינוי קטן → Stage → Commit.
- חפיפה: משלים את **מתקדם 09** (שם worktrees וענפים).

### B05. חשבון Anthropic / Claude ושימוש — רק מה שצריך להתחיל

- מטרה: התחברות לשירות Claude לפי המודל של **Claude Code** (מנוי Claude.ai / תמחור לפי תיעוד Anthropic בזמן הצילום); מתי שימוש "קל" מול "כבד"; איך לא לבזבז ריצות ארוכות בטעות.
- תוצר: הבנה היכן מנהלים מנוי/שימוש ומה משפיע על עלות.
- דמו: מסך חשבון או תמחור רשמי — בלי לנעול מספרים בדיבור.
- חפיפה: גרסה מקוצרת של **מתקדם 03–04** (שם Cursor ומודלים נרחבים יותר).

### B06. שאלה, Plan, ביצוע מבוקר, דיבוג (במונחי Claude Code)

- מטרה: להתאים את שלבי העבודה ל-**Claude Code ב-Cursor**: שאלה והבנה → **Plan mode** (תוכנית לאישור) → ביצוע עם אישורי פעולה / מצב רגיל → דיבוג כשמשהו נשבר.
- תוצר: עץ החלטה פשוט; אותה משימה קצרה בשלבים שונים.
- דמו: משימה אחת (למשל "הוסף כותרת לדף") — תכנון לפני עריכה, ואז שינוי מבוקר.
- חפיפה: מושגים מקבילים ל-**מתקדם 05** (שם מצבי Cursor); להתאים שמות כפתורים לגרסת ההרחבה בזמן ההקלטה.

### B07. הקשר פרויקט מינימלי (CLAUDE.md + זיכרון קצר)

- מטרה: שכבת הקשר מינימלית לפי מוסכמות **Claude Code** (למשל `CLAUDE.md`, והנחיות ב-`settings` / קבצי פרויקט לפי התיעוד המעודכן) ומסמך זיכרון קצר — **בלי** מטריצת MCP מלאה.
- תוצר: מסמך קצר: מה הפרויקט, מה אסור לשבור, איזה סגנון תשובה רצוי.
- דמו: יצירה עם עזרת הסוכן + עריכה ידנית קצרה.
- חפיפה: גרסה מצומצמת של **מתקדם 06** (שם לעיתים `.cursor/rules` וכו').

### B08. תכנון לפני קוד: דרישות, Plan, וסקיצת ממשק

- מטרה: Spec-driven קל: מסמך דרישות → שאלות הבהרה → תוכנית ב-**Plan** → wireframe בטקסט (ASCII) לפני קוד.
- תוצר: `PRD` חד-עמודי + סקיצת UI טקסטואלית מאושרת.
- דמו: תכנון ללא שינוי קבצים, ואז מעבר מבוקר לביצוע.
- חפיפה: משלים את **מתקדם 07** (לופ פיצ'ר מלא).

### B09. פרויקט א': אוטומציה אישית (נתונים / קבצים)

- מטרה: סקריפט קטן שרץ מקומית (למשל עיבוד CSV) עם ליווי הסוכן; טיפול בשגיאה ע"י הדבקת לוג לשיחה.
- תוצר: תוצר קובץ או דוח פשוט על המחשב.
- דמו: מהדרישות שב-B08 ועד הרצה במסוף.

### B10. פרויקט ב': אפליקציית רשת פשוטה + איטרציה חזותית

- מטרה: HTML/CSS/JS או מסלול פשוט — עם לoop קצר; משוב מצילום מסך בשיחה (אם זמין בהרחבה).
- תוצר: דף או אפליקציה שרצה מקומית.
- דמו: שימוש חוזר בתכנון לפני שינויי UI גדולים.

### B11. טעימה: MCP אחד ובטיחות הרשאות

- מטרה: מהו MCP; לפי התיעוד הרשמי — **הגדרת שרתים דרך ה-CLI תחילה**, וההרחבה משתמשת בהגדרות אלה; **שרת אחד** בלבד בבסיסי; עקרון least privilege. ראו [`claude-code-vs-code-extension-tips.md`](claude-code-vs-code-extension-tips.md).
- תוצר: MCP מופעל עם הסבר מה הוא רשאי לעשות.
- דמו: משימה אחת שמנצלת את הכלי החיצוני.
- חפיפה: כניסה רכה ל-**מתקדם 06 ו-10**.

### B12. גיבוי בענן: GitHub ו-push ראשון

- מטרה: חשבון GitHub; מאגר ראשון; חיבור הפרויקט המקומי ל-remote; **push ראשון** — הקוד נשמר בענן ונראה ב-GitHub.
- תוצר: מאגר GitHub עם לפחות commit אחד שנדחף; הבנה במילים פשוטות מה עושה `push`.
- דמו: Publish / הגדרת `origin` מ-Cursor (או זרימה מקבילה), בלי לערבב עדיין ספק פריסה נפרד.
- חפיפה: בסיס ל-**מתקדם 09–10**; הכנה ל-B13 ול-**מתקדם 13** (פריסה מקצועית).

### B13. פריסה ראשונה עם Vercel

- מטרה: לחבר מאגר GitHub אחד ל-**Vercel**, לקבל כתובת חיה, להבין **preview** מול **production** ברמת מתחילים, ולדעת איפה מגדירים סודות (לא בקוד ולא בצ'אט).
- תוצר: URL עובד (preview או production) לפרויקט מהמסלול (למשל מאחרי B10).
- דמו: ייבוא פרויקט מ-GitHub ל-Vercel, deploy, בדיקה קצרה בדפדפן.
- מחוץ לתחום: דומיין מותאם, build מורכב, rollback מעמיק — נשאר למסלול המתקדם.
- חפיפה: גרסה מצומצמת וברורה של רעיונות מ-**מתקדם 13**; המשך טבעי לקורס המתקדם.
