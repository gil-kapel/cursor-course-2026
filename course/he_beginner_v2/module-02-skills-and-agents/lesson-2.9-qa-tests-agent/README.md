## Syllabus topic

Test plans and focused automated tests: happy paths, edge cases, failures, and state transitions tied to acceptance criteria.

## Learning outcomes

- Write a **test plan** and **targeted automated tests** tied to acceptance criteria—not only the happy path.
- Cover edge cases, failure modes, and important **state transitions**.

## Where this sits

- **Phase:** Implementation — hardening behavior after features exist.
- **Prerequisite:** Defined acceptance criteria (from [2.2](../lesson-2.2-product-agent-prd/README.md) or equivalent) and code under test.
- **Next:** [Lesson 2.10 — Debug](../lesson-2.10-debug-agent/README.md) when things still break in practice.
