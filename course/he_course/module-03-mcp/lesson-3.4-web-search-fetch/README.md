## Syllabus topic

How to use web-search-style MCPs for live docs, library research, troubleshooting, and decision support, then generalize that same process to other MCPs the student wants to integrate into their own project workflow.

## Practical outcomes

- Use live search and fetch tools to answer a real build question with current information.
- Distinguish between MCPs that provide context, MCPs that manipulate project assets, and MCPs that connect to external systems.
- Build a simple decision rule for choosing which MCPs belong in a project and which ones are unnecessary noise.
- Learn a reusable project integration checklist for other MCPs: choose the job, connect the server, verify access, define prompts, validate output, and fold the result into the team's workflow.
- Leave the module with a short shortlist of additional MCPs the student may want to add later based on product, design, code, data, or deployment needs.

## Student task

- Pick one real question from your project that requires up-to-date information.
- Use Web Search or Fetch to answer only that one question.
- Turn the answer into one concrete next step: code change, note, TODO, or technical decision.
- Make a shortlist of 2 additional MCPs that might help your project later.
- For each one, write what job it would improve.

## Prompt

`You are a research assistant for a coding project. Goal: answer one real project question using live web information and turn it into one clear next step. Context: my question is [question]. Output: the best answer in simple language, the source I should trust most, and one concrete next action for the project. Constraints: focus on one question only, avoid extra browsing, and keep the answer practical.`
