## 00:00 – 01:30

בשיעור הזה אנחנו עוברים מתיאוריה לעבודה אמיתית בתוך Cursor. אם עד עכשיו חשבתם על הסוכן כמו על "מוח אחד", עכשיו הגיע הזמן לחשוב עליו כמו על **צוות**. לא כל אחד בצוות עושה הכל, וגם לא כל Agent אמור לעשות הכל.

המטרה המעשית שלנו פשוטה: עד סוף השיעור יהיו לכם לפחות שני Agents שונים עם תפקיד ברור, פתיח ברור, וגבולות קונטקסט ברורים.

---

## 01:30 – 03:30

הרעיון המרכזי פשוט: **Agent אחד = משימה אחת או תחום אחד**. למשל, Agent למוצר, Agent לארכיטקטורה, Agent לבנייה, Agent לביקורת. למה? כי כל Agent צובר הקשר. אם אותו Agent מקבל גם תכנון מוצר, גם שאלות עיצוב, גם דיבאג, וגם בדיקות, הקונטקסט שלו נהיה רועש, והתשובות מתחילות להתערבב.

במקום זה, לכל Agent מגדירים תפקיד ברור: מה הוא אמור לעשות, אילו קבצים בדרך כלל מצרפים לו, ואיזה פלט אתם מצפים לקבל.

החוק שכדאי לזכור הוא פשוט: **Task חדש = Context חדש**. לפעמים זה Agent חדש, ולפעמים זו שיחה חדשה עם אותו Agent. מה שלא עושים, זה לגרור שיחה ישנה רק כי היא כבר פתוחה.

---

## 03:30 – 05:00

דוגמאות: Product Agent מקבל רעיון ומחזיר PRD. Builder Agent מקבל PRD, ארכיטקטורה ו־Rules, ומחזיר diff קטן וסדר בדיקות. Review Agent מקבל diff, ומחזיר ממצאים - לא מחמאות כלליות.

ההבדל הזה חשוב כי **קונטקסט נשמר פר Agent ופר Task**. אם עברתם ממשימה של תכנון למימוש, אל תמשיכו באותו רצף אוטומטית. פתחו Agent אחר, או לפחות שיחה חדשה עם הקבצים הנכונים.

שימו לב גם לסכנה ההפוכה: לא צריך לפתוח עשרה Agents בלי סיבה. הרעיון הוא לא "יותר חלונות", אלא חלוקה ברורה של אחריות.

---

## 05:00 – 06:00

התרגיל שלכם כאן: ליצור לפחות שני Agents שונים, ולכתוב לכל אחד פתיח חד. לא "תעזור לי לבנות משהו", אלא פתיח שמגדיר תפקיד, מטרה, קונטקסט, פלט צפוי ומגבלות.

הבדיקה שלכם קלה: אם אתם יכולים להסתכל על שני ה־Agents ולהגיד במשפט אחד מה כל אחד מקבל ומה הוא מחזיר, עשיתם את העבודה נכון.

בשיעור הבא נוסיף את השכבה של ASM ונביא את הסקילים עצמם לפרויקט. נתראה ב־2.3.
