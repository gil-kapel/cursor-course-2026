# קורס Cursor בסיסי (עברית)

## מיקום במסלול הלמידה

זהו **מסלול הבסיס** של חבילת הקורס. הוא מיועד ללומדים **ללא רקע טכנולוגי פורמלי** שרוצים לעבוד עם סוכני AI בתוך **Cursor** (לא כתוסף נפרד ל-VS Code). לאחריו מומלץ להמשיך ל-[קורס Cursor מתקדם](../../he-advanced/docs/course-outline.md), שמרחיב את אותם נושאים לעומק מקצועי.

חפיפה מכוונת בין המסלולים מתועדת ב-[`cross-track-overlap.md`](cross-track-overlap.md).

## מטרת הקורס

להביא לומד מ**אפס ניסיון בסביבת פיתוח** עד ליכולת:

- לנווט ב-Cursor (תיקיות, עורך, צ'אט/אייג'נט, מסוף משולב) בביטחון.
- לראות שינויי קוד (diff) ולאשר או לדחות בצורה מבוקרת.
- לנהל **גיבוי מקומי** עם Git דרך הממשק הגרפי בלבד (נקודות שמירה), בלי תסריט של ענפים ומיזוגים.
- להריץ זרימת עבודה **Explore → Plan → Implement** עם מודלי Cursor.
- לבנות פרויקט קטן מקומי, להעלות אותו ל-GitHub, ולפרסם גרסה ראשונה בעזרת ספק אחד (Vercel) בשיעור ייעודי.

## קהל יעד

- מנהלי מוצר, שיווק, מחקר, יזמות — ללא רקע בקוד.
- משתמשים מנוסים ב-ChatGPT/Claude בדפדפן שרוצים **הקשר פרויקט מלא** וסוכן שעורך קבצים.
- כל מי שסובל מ"חרדת מסוף" או מ"פחד לשבור משהו" וצריך **רשת ביטחון** לפני שימוש באייג'נט.

## מה לא נלמד כאן (נשמר למסלול המתקדם)

- Worktrees, ענפים מקבילים וזרימות Git מורכבות.
- מטריצת MCPs מלאה, Skills מורכבים, ותוספים רחבים — רק טעימה.
- תמחור מודלים וניהול טוקנים לעומק (רק עקרונות).
- עבודת צוות, handoffs, pipeline פרסום/לוקליזציה של קורסים.

## מבנה הקורס

- פורמט מתוכנן: **13 סרטונים** (ניתן לפצל מודולים ארוכים לשני סרטונים בפועל).
- אורך מומלץ: **6 עד 12 דקות** לסרטון (קצב איטי יותר מהמסלול המתקדם).
- סגנון: דמו מעשי → הסבר קצר → צ'קליסט → משימת בית אחת.

## סטנדרט שיעור

- הנחיות הפקה: [`../../he-advanced/docs/recording-and-elevenlabs-guide.md`](../../he-advanced/docs/recording-and-elevenlabs-guide.md).
- גלוסריום מונחים: [`../../he-advanced/docs/hebrew-tts-glossary.md`](../../he-advanced/docs/hebrew-tts-glossary.md).

## סדר השיעורים

### B01. ממצרכן צ'אט למנצח תהליך

- מטרה: פרדיגמת סוכן מול צ'אט; חלון הקשר כשולחן עבודה מוגבל; למה עובדים **מקומית** ב-Cursor.
- תוצר: הלומד מבין שתפקידו לתאר דרישות, לאשר תוכניות ולנהל סיכון — לא "לזכור תחביר".
- דמו: השוואה קצרה: דפדפן מול חלון פרויקט ב-Cursor.

### B02. Cursor כחדר בקרה: התקנה וסיור בממשק

- מטרה: התקנה נקייה; אנלוגיות מרחביות — Explorer, עורך, פאנל AI, מסוף.
- תוצר: תיקיית פרויקט ריקה נפתחת ב-Cursor; קובץ טקסט ראשון.
- דמו: גרירת תיקייה, פתיחת מסוף משולב, שמירה.
- חפיפה: מקביל לרוחב של שיעורי **מתקדם 01–02** (כאן איטי ומילולי יותר).

### B03. המסוף בלי פחד: "הסבר לפני הרצה"

- מטרה: להפחית Terminal Anxiety — לפני פקודה לא מוכרת, לשאול ב-**Ask** מה היא עושה ואם היא הרסנית.
- תוצר: הרגל עבודה אחד (תבנית prompt) לפני הרצה.
- דמו: פקודה פשוטה אחת (למשל רשימת קבצים) עם הסבר מהסוכן.

### B04. Git כרשת ביטחון (נקודות שמירה)

- מטרה: Git **רק** כמכונת זמן / Save points; בלי ענפים, merge, origin בעומק.
- תוצר: Commit ראשון דרך **Source Control** ב-Cursor; הודעת commit משמעותית; הבנת צבעי סטטוס קבצים.
- דמו: שינוי קטן → Stage → Commit.
- חפיפה: משלים את **מתקדם 09** (שם worktrees וענפים).

### B05. חשבון, מנוי, ובחירת מודל — רק מה שצריך להתחיל

- מטרה: חשבון Cursor; מתי חינם מספיק; איך לא לבזבז בקשות כבדות בטעות.
- תוצר: בחירת מסלול בסיסי לפי שימוש.
- דמו: מסך מנוי + דוגמה אחת של משימה "קלה" מול "כבדה".
- חפיפה: גרסה מקוצרת של **מתקדם 03–04**.

### B06. מודים: Ask, Plan, Agent, Debug

- מטרה: מתי לשאול, מתי לתכנן, מתי לבצע, מתי לחקור תקלה.
- תוצר: עץ החלטה פשוט על המקרר; אותה משימה קצרה בשני מודים שונים.
- דמו: משימה אחת (למשל "הוסף כפתור") ב-Ask → Plan → Agent.
- חפיפה: ישיר מול **מתקדם 05** (שם קצב מהיר ויותר דוגמאות).

### B07. הקשר פרויקט מינימלי (Rules + זיכרון קצר)

- מטרה: קובץ כללים/הקשר פרויקט (למשל `.cursor/rules` ו/או `AGENTS.md` לפי המוסכמות של הפרויקט) — **בלי** מטריצת MCP מלאה.
- תוצר: מסמך קצר: מה הפרויקט, מה אסור לשבור, איזה סגנון תשובה רצוי.
- דמו: יצירה עם עזרת הסוכן + עריכה ידנית קצרה.
- חפיפה: גרסה מצומצמת של **מתקדם 06**.

### B08. תכנון לפני קוד: דרישות, Plan mode, וסקיצת ממשק

- מטרה: Spec-driven קל: מסמך דרישות → שאלות הבהרה → תוכנית ב-**Plan** → wireframe בטקסט (ASCII) לפני קוד.
- תוצר: `PRD` חד-עמודי + סקיצת UI טקסטואלית מאושרת.
- דמו: Plan mode ללא שינוי קבצים, ואז מעבר מבוקר ל-Agent.
- חפיפה: משלים את **מתקדם 07** (לופ פיצ'ר מלא).

### B09. פרויקט א': אוטומציה אישית (נתונים / קבצים)

- מטרה: סקריפט קטן שרץ מקומית (למשל עיבוד CSV) עם ליווי האייג'נט; טיפול בשגיאה ע"י הדבקת לוג ל-Ask/Debug.
- תוצר: תוצר קובץ או דוח פשוט על המחשב.
- דמו: מהדרישות שב-B08 ועד הרצה במסוף.

### B10. פרויקט ב': אפליקציית רשת פשוטה + איטרציה חזותית

- מטרה: HTML/CSS/JS או מסלול מודרני פשוט — עם לoop קצר; משוב מצילום מסך בצ'אט (אם זמין במוצר).
- תוצר: דף או אפליקציה שרצה מקומית.
- דמו: שימוש חוזר ב-Plan לפני שינויי UI גדולים.

### B11. טעימה: MCP אחד ובטיחות הרשאות

- מטרה: מהו MCP; התקנה/הפעלה של **שרת אחד** רלוונטי; עקרון least privilege.
- תוצר: MCP מופעל עם הסבר מה הוא רשאי לעשות.
- דמו: משימה אחת שמנצלת את הכלי החיצוני.
- חפיפה: כניסה רכה ל-**מתקדם 06 ו-10**.

### B12. גיבוי בענן: GitHub ו-push ראשון

- מטרה: חשבון GitHub; מאגר ראשון; חיבור הפרויקט המקומי ל-remote; **push ראשון** — הקוד נשמר בענן ונראה ב-GitHub.
- תוצר: מאגר GitHub עם לפחות commit אחד שנדחף; הבנה במילים פשוטות מה עושה `push`.
- דמו: Publish / הגדרת `origin` מ-Cursor (או זרימה מקבילה), בלי לערבב עדיין ספק פריסה נפרד.
- חפיפה: בסיס ל-**מתקדם 09–10**; הכנה ל-B13 ול-**מתקדם 13** (פריסה מקצועית).

### B13. פריסה ראשונה עם Vercel

- מטרה: לחבר מאגר GitHub אחד ל-**Vercel**, לקבל כתובת חיה, להבין **preview** מול **production** ברמת מתחילים, ולדעת איפה מגדירים סודות (לא בקוד ולא בצ'אט).
- תוצר: URL עובד (preview או production) לפרויקט מהמסלול (למשל מאחרי B10).
- דמו: ייבוא פרויקט מ-GitHub ל-Vercel, deploy, בדיקה קצרה בדפדפן.
- מחוץ לתחום: דומיין מותאם, build מורכב, rollback מעמיק — נשאר למסלול המתקדם.
- חפיפה: גרסה מצומצמת וברורה של רעיונות מ-**מתקדם 13**; המשך טבעי לקורס המתקדם.

- שיעורים: B01–B04
- תוצר: Cursor פתוח, מסוף לא מאיים, Git כ-save points

### Arc B — שיטת עבודה עם AI

- שיעורים: B05–B08
- תוצר: מודים + הקשר פרויקט מינימלי + תכנון לפני קוד

- שיעורים: B09–B10
- תוצר: שני תוצרים מקומיים חיים

- שיעורים: B11–B13

## תוצרים צפויים מהלומד בסוף המסלול

- להסביר במילים פשוטות מה הסוכן רשאי לעשות בפרויקט ומה לא.
- לבחור מוד נכון לשלב המשימה.
- לשמור מצב עבודה ב-Git לפני ניסויים מסוכנים.
- לבנות תכנית ואז לממש בפרוסות עם אימות.
- להריץ פרויקט קטן מקומית, לדחוף ל-GitHub, ולפרסם גרסה ראשונה לענן עם Vercel.
