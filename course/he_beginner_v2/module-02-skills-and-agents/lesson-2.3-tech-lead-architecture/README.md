## Syllabus topic

Translating a PRD into technical decisions: stack, boundaries, folder layout, APIs, and implementation slices the team can execute without re-litigating the spec.

## Learning outcomes

- Turn a PRD into **concrete technical choices** and **execution slices** (not a generic “we’ll use React”).
- Document boundaries so UX (2.4) and UI (2.5) can argue about screens, not about stack.
