# Script Master: B06 Modes — Ask, Plan, Agent, Debug

## Metadata

- Lesson ID: B06
- Title: מודים: Ask, Plan, Agent, Debug
- Target duration: 09:00
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

להבין מתי להשתמש בכל mode; להריץ אותה משימה קצרה ב-Ask ואז ב-Plan וב-Agent כדי להרגיש את ההבדל.

## Examples (דוגמאות קונקרטיות)

### אותה משימה בארבעה מצבים

- `Ask`: "איפה בקוד מוגדרת כותרת הדף, ומה הסיכון בשינוי?"
- `Plan`: "תן תוכנית קצרה ב-3 צעדים להוספת כותרת בולטת בלי לבצע שינוי."
- `Agent`: "יישם רק את שלב 1: הוסף כותרת לדף הבית ואל תשנה עיצוב נוסף."
- `Debug`: "הוספת הכותרת שברה את הפריסה. מה שלוש סיבות סבירות ומה בודקים קודם?"

### כללי אצבע מהירים

- אם אתם עוד מבינים את הבעיה: `Ask`
- אם יש כמה כיוונים או tradeoffs: `Plan`
- אם הדרישה ברורה ומצומצמת: `Agent`
- אם משהו נשבר או לא מובן: `Debug`

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** להשתמש באותה משימה קטנה לאורך כל השיעור כדי שההבדל בין המצבים יהיה ברור.

### משימה 1 — לנסח בקשה ל-Ask (אחרי S02)

- **עצירה:** אחרי הסבר Ask.
- **מה לעשות:** לבחור משימה קטנה אחת ולכתוב ניסוח שמתאים ל-Ask בלבד, בלי לבקש שינוי קבצים.
- **איך לוודא הצלחה:** הניסוח מבקש הבנה, הסבר, או מיקום בקוד, ולא "תבצע".

### משימה 2 — לנסח בקשה ל-Plan (אחרי S03)

- **עצירה:** אחרי הסבר Plan.
- **מה לעשות:** לקחת את אותה משימה ולנסח אותה מחדש כך שתבקש תוכנית שלבים בלי עריכה בפועל.
- **איך לוודא הצלחה:** יש בבקשה מילים כמו "תוכנית", "שלבים", או "אילו קבצים", ואין בקשה לבצע.

### משימה 3 — לעבור לביצוע מבוקר או Debug (אחרי S04-S05)

- **עצירה:** אחרי Agent ו-Debug.
- **מה לעשות:** לבחור אחד: או לנסח בקשת Agent מוגבלת לשלב אחד, או לנסח בקשת Debug על תקלה אמיתית/מדומה.
- **איך לוודא הצלחה:** בקשת Agent כוללת גבול ברור; בקשת Debug מתחילה בהבנה ולא ב"תתקן הכול".
