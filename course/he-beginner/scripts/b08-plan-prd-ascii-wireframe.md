# Script Master: B08 Plan Before Code — PRD and ASCII Wireframe

## Metadata

- Lesson ID: B08
- Title: תכנון לפני קוד: דרישות, Plan, וסקיצת ממשק
- Target duration: 09:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

Spec-driven קל: PRD חד-עמודי, שאלות הבהרה, תוכנית ב-Plan mode, wireframe ASCII לפני מימוש ב-Agent.

## Examples (דוגמאות קונקרטיות)

### דוגמאות ל-PRD קטן וטוב

- "אפליקציה קטנה למעקב הוצאות יומיות."
- "דף פשוט שמרכז לינקים לצוות."
- "סקריפט שמסכם CSV לקובץ טקסט."

### מה חייב להופיע ב-PRD הבסיסי

- למי זה מיועד.
- מה הבעיה שנפתרת.
- מה חייב להיות בגרסה הראשונה.
- מה בכוונה לא נכנס עכשיו.

### דוגמה קצרה ל-ASCII wireframe

- `[כותרת]`
- `[שדה חיפוש] [כפתור]`
- `[רשימת תוצאות]`

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** השיעור הזה בנוי בעצירות טבעיות בין מסמך, שאלות, תוכנית, וסקיצה.

### משימה 1 — PRD של עמוד אחד (אחרי S02)

- **עצירה:** אחרי תבנית ה-PRD.
- **מה לעשות:** לכתוב גרסה ראשונית של PRD קצר עם ארבעה חלקים: משתמש, בעיה, חובה בגרסה ראשונה, לא בשלב זה.
- **איך לוודא הצלחה:** אפשר לקרוא את ה-PRD בפחות מדקה, והוא לא גולש למסמך ארוך.

### משימה 2 — שאלות הבהרה (אחרי S03)

- **עצירה:** אחרי בקשת שאלות ההבהרה ב-Ask.
- **מה לעשות:** לבקש מ-Cursor ארבע שאלות הבהרה, ולענות לפחות על שתיים מהן.
- **איך לוודא הצלחה:** ה-PRD עודכן או השתפר בעקבות התשובות, לא נשאר אותו טקסט בדיוק.

### משימה 3 — סקיצה ואישור שלב ראשון (אחרי S05-S06)

- **עצירה:** אחרי ה-wireframe והמעבר מ-Plan ל-Agent.
- **מה לעשות:** לצייר מסך אחד ב-ASCII, ואז לנסח בקשת Agent שמאשרת רק את שלב 1.
- **איך לוודא הצלחה:** מישהו אחר יכול להבין מה יופיע במסך, והבקשה כוללת גבול ברור כמו "עצור אחרי יצירת המבנה".
