# חפיפה בין מסלול בסיסי למסלול מתקדם

מסמך זה מגדיר **חפיפה מכוונת אך מוגבלת**. **המסלול הבסיסי** משתמש ב-**Cursor** עם ההרחבה הרשמית **Claude Code for VS Code** (Anthropic) כסוכן המרכזי על הפרויקט. **המסלול המתקדם** מניח **Cursor** ומתמקד ב-delivery מקצועי בתוך repo אמיתי (כולל כלי Cursor נוספים, worktrees, MCP עומק, AWS). העקרונות (תכנון, diff, Git, בטיחות, פריסה) זהים ברובם; **במתקדם** העומק והאילוצים גדלים.

## עקרון

| במסלול הבסיסי (Cursor + Claude Code) | במסלול המתקדם (Cursor) |
| -------------------------------------- | ---------------------- |
| קצב איטי, מטאפורות, בניית ביטחון | קצב מקצועי, אילוצים, איכות ושחרור |
| Greenfield קטן ומבוקר | Codebase קיים, feature אמיתי, סיכון אמיתי |
| סוכן ראשי: הרחבת Claude Code | Cursor מלא: מצבים, rules, MCP מורכב, צוות |
| CLI Claude Code — רק הקדמה ב-B01 | אפשר שימוש ב-CLI במסגרות מתקדמות לפי צורך |

## מה חופף באמת

החפיפה הישירה היא בעיקר ביסודות:

- מה הסוכן יודע לעשות ומה לא
- שלבי שאלה / תכנון / ביצוע / דיבוג (שמות יכולים להשתנות בין Claude Code למצבי Cursor המובנים)
- הגדרת context בסיסי לפרויקט (`CLAUDE.md` וכו' מול `.cursor/rules` במתקדם)
- בטיחות בסיסית עם Git ו-MCP

במסלול המתקדם היסודות האלה נדחסים ל-**Bridge מהיר** בשיעור 02 ולא תופסים שוב את מרכז הבמה.

## המלצה למדריך

- מסלול **בסיסי → מתקדם**: הלומד כבר ב-Cursor עם Claude Code; במתקדם להדגיש **הרחבת ארגז הכלים** (לא החלפת עורך).
- מסלול **רק מתקדם** למפתחים ותיקים: אפשר לדלג על בסיסי אם כבר שולטים ב-Claude Code וב-Git — אך לוודא הבנת פריסה ראשונה.
- אם לומד חזק ב-Claude Code אבל חלש ב-delivery, לא לדלג על `04–13` במתקדם.
