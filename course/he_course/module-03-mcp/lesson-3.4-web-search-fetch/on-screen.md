## 00:00 – 01:30

פתיחה עם שאלה אמיתית מהפרויקט:
- integration question
- library setup question
- troubleshooting question

מראים שהמידע הרצוי לא יושב בתוך הקוד עצמו אלא מחוץ לפרויקט.

Overlay: **Use live information when the project needs live information**.

---

## 01:30 – 03:00

הדגמה של Web Search / Fetch:
- חיפוש שאלה אחת ברורה
- פתיחת מקור רלוונטי
- סיכום התשובה
- המרה ל-next step ברור

עדיף להראות דוגמה עדכנית מדוקומנטציה ולא חיפוש כללי מדי.
שורת דגש: **שאלה אחת בכל פעם**.

---

## 03:00 – 04:30

מסך whiteboard קצר עם 3 categories:
- Context MCPs
- Asset / workflow MCPs
- External-system MCPs

ליד זה 3 decision filters:
- repeated bottleneck
- trusted source
- useful output

דוגמאות קצרות ליד כל קטגוריה:
- Stitch / docs
- Figma / Browser
- GitHub / database

---

## 04:30 – 06:00

מסך סיום עם checklist לשילוב MCP חדש:
- Pick one job
- Connect the server
- Verify access
- Define one prompt
- Save one artifact
- Keep it only if it improves the workflow

שורת סיום למתחילים: **Start with 1-2 MCPs, not 7**.

כותרת סיום: **Module 3 complete — MCP is useful only when it changes the project workflow**.
