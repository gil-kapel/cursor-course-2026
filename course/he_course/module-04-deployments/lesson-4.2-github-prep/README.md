## Syllabus topic

Prepare the project for public or team-facing work: clean the repo, confirm what must stay out of git, make the first useful README, and push with a workflow that beginners can actually repeat safely.

## Practical outcomes

- Check what is safe to push and what must stay local.
- Confirm `.env` and local DB artifacts are not accidentally tracked.
- Create a small README that helps the next person run the app.
- Make one clean commit and push the repo without panic.
- Understand when `main` is fine and when a branch is the safer move.
