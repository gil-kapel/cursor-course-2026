## Running example

Task management app for teams — continued from lessons 2.4–2.5.

## Scope

Using Ask, Plan, and Agent modes to build a UX flow for the task management app. Ask identifies friction. Plan structures the flow. Agent saves `docs/ux-flows.md`.

## Verify

- `docs/ux-flows.md` covers the main path: login, empty state, add task, update status, delete.
- Each step includes loading, empty, and error states.
- You can explain why you used each mode and what it gave you.
- The UX skill asked clarifying questions before writing the final flow.
