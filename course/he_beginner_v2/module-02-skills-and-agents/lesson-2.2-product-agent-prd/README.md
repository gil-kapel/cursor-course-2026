## Syllabus topic

Turning a vague idea into a structured PRD: problem, users, scope, flows, metrics, acceptance criteria, and open questions—using the bundled product skill as the default workflow.

## Learning outcomes

- Produce a **PRD-shaped** artifact (sections, scope boundaries, measurable acceptance criteria) instead of a vague brainstorm.
- Know what to **attach in chat (`@`)** when handing off to architecture (Lesson 2.3).
