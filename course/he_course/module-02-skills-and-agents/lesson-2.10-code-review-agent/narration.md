# שיעור 2.10 (ElevenLabs / עברית)
---

## 00:00 – 01:00

הקוד רץ, הכל נראה בסדר. עצרו רגע.

עכשיו מפעילים **סוכן Code Review**. לא כדי לקבל מחמאות — אלא לבדוק שלא הכנסנו טעויות בזמן הפיתוח המהיר.

## 01:00 – 02:30

מבקש מהסוכן לעבור על ה-diff — רק על מה שהשתנה.

הסוכן מחלק את ההערות לפי חומרה: High, Medium, Low. הוא מחפש לוגים שנשארו, חוסר טיפול בשגיאות, שימוש לא נכון בטיפוסים.

אם הסוכן מציע לשכתב את כל האפליקציה — עוצרים. סקירה צריכה להיות ממוקדת בשינוי הנוכחי.

## 02:30 – 03:45

לוקח הערה אחת משמעותית ומתקן אותה במקום. לפעמים טיפול נכון בשגיאה או הסרת קוד מבלבל עושים את ההבדל.

## 03:45 – 05:00

שולח: "מה הכי דחוף לתקן לפני merge?"

זה עוזר לסנן רעש ולהתמקד: באגים לוגיים, דליפות נתונים, בעיות ביצועים.

## 05:00 – 06:00

לסיכום: סקרנו, תיקנו, ואישרנו.

בשיעור הבא, 2.11, מפעילים סוכן QA — תוכנית בדיקות וטסט אוטומטי ראשון.
