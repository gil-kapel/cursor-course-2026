## Syllabus topic

What a **local database** actually is, when to use **SQLite** versus a simple file like JSON, and how to build a tiny practical data layer that your app can read from and write to before moving anything to the cloud.

## Practical outcomes

- Understand local DB as a file-backed data store that lives on your machine.
- Use a simple decision rule: JSON for a tiny settings file, SQLite for real app data.
- Define one small schema from a real screen or feature before touching code.
- Create a local DB, add seed data, run a read/write flow, and verify persistence.
- Understand what stays the same and what changes when moving to Supabase next.
