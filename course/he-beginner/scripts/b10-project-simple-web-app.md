# Script Master: B10 Project B — Simple Web App + Visual Iteration

## Metadata

- Lesson ID: B10
- Title: פרויקט ב': אפליקציית רשת פשוטה + איטרציה חזותית
- Target duration: 10:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

לבנות דף או אפליקציית רשת מינימלית (HTML/CSS/JS או מסגרת פשוטה); לולאת משוב עם Plan לפני שינויי UI גדולים; צילום מסך לצ'אט אם המוצר תומך.

## Examples (דוגמאות קונקרטיות)

### רעיונות טובים לפרויקט ווב ראשון

- דף נחיתה אישי עם כותרת וכפתור
- רשימת משימות קטנה עם פעולה אחת
- דף לינקים פשוט עם מצב בהיר/כהה

### פידבק ויזואלי קצר שאפשר לתת בפאנל Claude Code

- "הכותרת גדולה מדי."
- "הכפתור קרוב מדי לשוליים."
- "המרווח בין הכרטיסים קטן מדי."

### גבולות טובים לאיטרציה הראשונה

- מסך אחד
- אלמנט אינטראקטיבי אחד
- בלי auth, backend, או database

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** לשמור על תחושת ניצחון מהירה. עדיף דף קטן שעובד מאשר פרויקט גדול שלא נפתח.

### משימה 1 — מתרגמים סקיצה לקבצים (אחרי S02)

- **עצירה:** אחרי המעבר מ-ASCII לקבצים.
- **מה לעשות:** לכתוב אילו 2-4 קבצים נדרשים לגרסה הראשונה.
- **איך לוודא הצלחה:** רשימת הקבצים קצרה וברורה, ולא מתחילה לגדול לפרויקט מורכב.

### משימה 2 — לפתוח בדפדפן ולבדוק פעולה אחת (אחרי S03)

- **עצירה:** אחרי פתיחת הפרויקט בדפדפן.
- **מה לעשות:** לפתוח את הדף ולוודא שאלמנט אחד לפחות מופיע או מגיב.
- **איך לוודא הצלחה:** אתם רואים מסך עובד, לא רק קבצים בעורך.

### משימה 3 — תיקון UI קטן ושמירה יציבה (אחרי S04-S07)

- **עצירה:** אחרי לולאת הפידבק החזותי, ה-Plan הקצר, וההמלצה לבצע commit.
- **מה לעשות:** לבקש תיקון UI קטן אחד בלבד, וכשהמסך נראה סביר ועובד לבצע commit.
- **איך לוודא הצלחה:** הבקשה ממוקדת לשינוי קטן אחד, ויש נקודת שמירה שאפשר לחזור אליה לפני עוד ניסויים.

### S01 | 00:00–00:40 | Hook

- **מה על המסך:** דפדפן עם דף פשוט (כותרת + כפתור או רשימה קצרה).
- **פעולות:** רענון או לחיצה אחת להדגמת אינטראקציה.
- **קריינות:** "עכשיו מוסיפים שכבה שאפשר לראות: משתמש פותח דף, לוחץ, רואה צבעים. עדיין בפרויקט קטן — לא סטארטאפ."
- **אם לא מספיק:** שקופית עם צילום מסך סטטי; בלי דפדפן חי.

### S02 | 00:40–01:50 | From ASCII to Files

- **מה על המסך:** קובץ עם wireframe מ-B08; מעבר ל־Plan בפאנל; רשימת קבצים מוצעת (`index.html` וכו').
- **פעולות:** גלילה בין סקיצה לתוכנית; אישור שלב ראשון.
- **קריינות:** "לוקחים את הסקיצה מהטקסט. ב-**Plan** שואלים אילו קבצים נדרשים לגרסה הראשונה. רק אחרי אישור תוכנית — ביצוע עם diff."
- **עצירת כיתה:** **משימה 1** — מתרגמים סקיצה לקבצים.
- **אם לא מספיק:** הציגו רשימת קבצים מוכנה מראש; דלגו על יצירה חיה.

### S03 | 01:50–03:05 | Open in Browser

- **מה על המסך:** קליק ימני על `index.html` / Live Server / `Open with…`; הדף נפתח בכרום או בדפדפן מובנה.
- **פעולות:** פתיחה אחת; זום על אלמנט אחד בדף.
- **קריינות:** "רואים את התוצאה בדפדפן. אם אין שרת — לפעמים מספיק לפתוח HTML; אם הסוכן מציע שרת מקומי, חוזרים ל-B03 לפני הרצה."
- **עצירת כיתה:** **משימה 2** — לפתוח בדפדפן ולבדוק פעולה אחת.
- **אם לא מספיק:** הדגימו רק את הנתיב בתפריט "פתח בדפדפן"; אמרו שהתלמידים יאמתו לבד.

### S04 | 03:05–04:15 | Visual Feedback Loop

- **מה על המסך:** צילום מסך של הדף; בפאנל — צירוף תמונה דרך בורר הקבצים; הודעה קצרה ("הכפתור למטה", "הכותרת גדולה מדי").
- **פעולות:** שמירת PNG → צירוף → שליחה; גלילה בתשובה.
- **קריינות:** "**טיפ מהתיעוד הרשמי:** בהרחבה אפשר לצרף **קבצים ותמונות** דרך בורר הקבצים של המערכת. מצלמים את הדף, מצרפים, כותבים: הכפתור צריך לרדת שורה, הכותרת גדולה מדי — חוסך פסקאות."
- **אם לא מספיק:** אין צירוף תמונה בגרסה שלכם — הראו רק את בורר הקבצים ואמרו "כאן מצרפים צילום מסך"; המשיכו בטקסט בלבד.

### S05 | 04:15–05:25 | Plan Before Big UI Rewrite

- **מה על המסך:** Plan קצר: עמודות "נשאר" / "משתנה"; או רשימת נקודות לפני שינוי UI.
- **פעולות:** עריכת שורה אחת בתוכנית; ללא שינוי קוד בזמן הסצנה.
- **קריינות:** "לפני 'תעצב מחדש הכול' — Plan קצר: מה משתנה, מה נשאר. מונע בלגן בקבצים."
- **עצירת כיתה (הכנה):** **משימה 3** — תיקון UI קטן ושמירה יציבה.
- **אם לא מספיק:** שקופית עם שתי עמודות; 45 שניות קריינות בלבד.

### S06 | 05:25–06:35 | Dark Mode Toggle (Optional Scope)

- **מה על המסך:** אותו דף במצב בהיר ואז כהה (מעבר מהיר) או טוגל ב-UI.
- **פעולות:** לחיצה על טוגל אחד; לא לערבב עם שינויי פריסה גדולים באותה סצנה.
- **קריינות:** "אם ב-PRD היה dark mode — מוסיפים כשלב שני אחרי שהמבנה עובד. לא לערבב הכול בבת אחת."
- **אם לא מספיק:** דלגו על dark mode; אמרו שזה שלב שני אופציונלי והציגו את הדף הבהיר בלבד.

### S07 | 06:35–07:35 | Commit After Stable Screen

- **מה על המסך:** Git ב-Cursor — הודעת commit קצרה; או ממשק Source Control עם קבצים מסומנים.
- **פעולות:** Commit אחד מודגם (או הסבר על הכפתור בלי commit אמיתי אם אין זמן).
- **קריינות:** "כשמסך נראה סביר — commit. אז ממשיכים לשחק."
- **עצירת כיתה:** **משימה 3** — תיקון UI קטן ושמירה יציבה.
- **אם לא מספיק:** רק משפט "נקודת שמירה לפני ניסויים"; חזרה ל-B04 אם צריך פירוט.

### S08 | 07:35–08:45 | Debug When Layout Breaks

- **מה על המסך:** DevTools בדפדפן — טאב Console עם שורת שגיאה אחת; צילום מסך של הקונסול בפאנל.
- **פעולות:** פתיחת DevTools; גלילה קצרה; העתקת שגיאה (אופציונלי).
- **קריינות:** "אם משהו 'נשבר', אפשר לצלם קונסול ולצרף לפאנל — מצב דיבוג. לא חייבים להבין כל שגיאת JavaScript."
- **אם לא מספיק:** צילום מסך סטטי של קונסול; בלי לפתוח כלים חיים.

### S09 | 08:45–09:45 | חפיפה למתקדם (MCP דפדפן)

- **מה על המסך:** כרטיס "מסלול מתקדם" + אייקון דפדפן/MCP.
- **פעולות:** דיבור בלבד או שקופית אחת.
- **קריינות:** "במסלול המתקדם נלמד MCP לדפדפן לעומק. כאן עיניים אנושיות וצילום מסך מספיקים."
- **אם לא מספיק:** קצרו ל־40 שניות; אל תפתחו B11 טכני כאן.

### S10 | 09:45–10:30 | סיום ו-B11

- **מה על המסך:** כותרת B11 + אייקון "שקע אחד" / MCP.
- **פעולות:** מעבר לשקופית סיום.
- **קריינות:** "בשיעור הבא נוסיף כלי חיצוני אחד דרך MCP — בזהירות ובהרשאות מינימליות."
- **אם לא מספיק:** קצרו S09; השאירו זמן ל-Action Item (צילום מסך אחד לצ'אט).
