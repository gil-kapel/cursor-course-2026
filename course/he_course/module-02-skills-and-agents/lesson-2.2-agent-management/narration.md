# Lesson 2.2 — Voiceover (ElevenLabs / Hebrew)

Plain paragraphs for TTS. Section headers match on-screen beats. Trim pauses in ElevenLabs after timing the cut.

---

## 00:00 – 01:30

בשיעור הזה אנחנו עוברים מתיאוריה לעבודה אמיתית בתוך Cursor. אם עד עכשיו חשבתם על הסוכן כמו על "מוח אחד", עכשיו הגיע הזמן לחשוב עליו כמו על **צוות**. לא כל אחד בצוות עושה הכל, וגם לא כל Agent צריך לעשות הכל.

---

## 01:30 – 03:30

הרעיון המרכזי פשוט: **Agent אחד = משימה אחת או תחום אחד**. למשל, Agent למוצר, Agent לארכיטקטורה, Agent לבנייה, Agent לביקורת. למה? כי כל Agent צובר הקשר. אם אותו Agent מקבל גם תכנון מוצר, גם שאלות עיצוב, גם דיבאג, וגם בדיקות, הקונטקסט שלו נהיה רועש, והתשובות מתחילות להתערבב.

במקום זה, לכל Agent מגדירים תפקיד ברור: מה הוא אמור לעשות, אילו קבצים בדרך כלל מצרפים לו, ואיזה פלט אתם מצפים לקבל.

---

## 03:30 – 05:00

דוגמאות: Product Agent מקבל רעיון וצריך להחזיר PRD. Builder Agent מקבל PRD, ארכיטקטורה ו־rules וצריך להחזיר diff קטן וסדר בדיקות. Review Agent מקבל diff וצריך להחזיר findings, לא מחמאות כלליות.

ההבדל הזה חשוב כי **קונטקסט נשמר פר Agent ופר Task**. אם עברתם ממשימה של תכנון למימוש, אל תמשיכו באותו רצף אוטומטית. פתחו Agent אחר, או לפחות שיחה חדשה עם הקבצים הנכונים.

---

## 05:00 – 06:30

התרגיל שלכם כאן: ליצור לפחות שני Agents שונים ולכתוב לכל אחד פתיח חד. לא "תעזור לי לבנות משהו", אלא "אתה Product Agent. קלט: רעיון + @brief. פלט: PRD קצר עם acceptance criteria". ברגע שהפתיח חד, הסוכן מבין טוב יותר את המסגרת.

בשיעור הבא נוסיף את השכבה של ASM ונביא את הסקילים עצמם לפרויקט. נתראה ב־2.3.
