## Lesson Goal

לפתוח ב-**Claude Code ב-CLI** (שורת פקודה): מה זה נותן ומה חסר למתחילים. אחר כך להסביר למה **אותו מוצר Anthropic** בתוך עורך — **Visual Studio Code או Cursor** — עדיף לרוב עבודת הפרויקט: diff, אישורים, עץ קבצים, מסוף משולב. במסלול הזה נעבוד ב-**Cursor** עם ההרחבה הרשמית **Claude Code for VS Code**. לסיים בכיול ציפיות: הלומד מנהל תהליך וסיכון, לא שולף תחביר.

## Examples (דוגמאות קונקרטיות)

### CLI Claude Code — מתי זה הגיוני

- שרת מרוחק, סקריפטים, אוטומציה, זרימה שמתחילה ונגמרת בטרמינל.
- "אני כבר נוח ב-shell ורוצה את אותו סוכן בלי לפתוח חלון עורך."

### למה בעורך (VS Code / Cursor) זה לרוב נוח יותר למתחילים

- **Diff ויזואלי** לפני שמקבלים שינוי בקובץ.
- **עץ הפרויקט** זהה לסוכן ולכם — פחות בלבול "איזה קובץ נגעו בו".
- **מסוף משולב** — אותו חלון; אפשר לשאול את הסוכן מה הפקודה עושה לפני Enter.
- **Cursor** הוא מבוסס VS Code, ולכן אפשר להתקין את **Claude Code for VS Code** מממשק ההרחבות — זה סטאק הקורס הבסיסי.

### מה מתאים יותר לצ'אט בדפדפן

- "תסכם לי את המאמר הזה בשלוש נקודות."
- "תכתוב לי מייל ללקוח בטון רגוע."

### מה מתאים יותר ל-Claude Code על תיקיית פרויקט ב-Cursor

- "תעבור על הקבצים בתיקייה הזאת ותציע תוכנית לשיפור דף הבית."
- "תעדכן רק את הטקסט בכפתור בקובץ הפתוח ואל תיגע בשום קובץ אחר."

### דוגמאות לציפייה נכונה מהלומד

- לא: "תבנה הכול לבד."
- כן: "הצע תוכנית קצרה, עצור לפני שינוי, והסבר מה תערוך."

### S01 | 00:00–00:40 | Hook — שלוש דלתות ל-Claude

- **מה על המסך:** פיצול או שלושה פאנלים סטטיים: (1) טאב דפדפן עם צ'אט; (2) חלון טרמינל כהה עם שורת הפקדה בלבד; (3) **Cursor** עם סרגל צד ואייקון **Spark** (או מלבן עם כיתוב "Claude Code"). אפשר שקופית עם שלוש אייקוניות וכיתוב בעברית.
- **פעולות במסך:** אין קליקים מהירים — רק זום קל על כל אזור בנפרד (3–4 שניות כל אחד) או מעבר חלק בין שלושת המסכים.
- **קריינות:** "יש כמה דרכים לדבר עם קלוד על קוד. היום נסדר את השולחן: מה זה **Claude Code ב-CLI**, למה הרבה אנשים עוברים ל-**עורך עם ההרחבה הרשמית** — ואיך זה מתחבר לצ'אט בדפדפן שכבר מכירים."
- **אם לא מספיק:** אין Cursor מותקן בצילום — השתמשו בשקופית או בצילום מסך סטטי מוכן מראש; אין צורך בפרויקט אמיתי.

### S02 | 00:40–01:45 | מה זה CLI Claude Code

- **מה על המסך:** טרמינל עם פרומפט בתיקייה (נתיב מסתיים בשם תיקיית דמה). אופציונלי: הקלדה איטית של `claude` ואנטר — **בלי** הצגת API keys, **בלי** הדבקת טקסט ארוך. חלופה: צילום מטושטש מדף התיעוד של Anthropic.
- **פעולות במסך:** עצירה על שורת הפקודה; הדגשה ויזואלית (מסגרת) סביב אזור הפלט הטקסטואלי בלבד — לא על חלון diff.
- **קריינות:** "**CLI** זה שורת פקודה: מריצים את כלי Claude Code מהמסוף, לרוב בתיקיית הפרויקט. זה חזק — סקריפטים, שרתים, אוטומציה. אבל למתחילים זה לעיתים **פחות שקוף**: פחות קל לראות בבת אחת *מה* השתנה בקבצים, וקל יותר 'לאבד שליטה' בלי diff ברור על המסך."
- **אם לא מספיק:** אין CLI מותקן — הציגו רק **אנימציה** או **שקופית** עם טרמינל; אל תתקינו בשידור חי.

### S03 | 01:45–03:00 | למה בעורך (VS Code / Cursor) עדיף לרוב

- **מה על המסך:** מסך מפוצל: משמאל טרמינל עם טקסט גולמי; מימין אותו רעיון ב-Cursor — **עץ קבצים**, קובץ פתוח, ופאנל עם **הדגשת שינוי** (שורות ירוקות/אדומות או גרסת diff של ההרחבה).
- **פעולות במסך:** גרירה קצרה של גבול הפיצול; לחיצה אחת על שורה ב-diff להרחבה (אם זמין).
- **קריינות:** "אותו מנוע עבודה של Anthropic מגיע גם כ-**הרחבה ל-VS Code**: **Claude Code for VS Code**. רואים את עץ הקבצים, מקבלים **הצעות שינוי עם diff**, מאשרים או דוחים, והמסוף באותו חלון. **Cursor** מבוסס על VS Code — אפשר להתקין שם את אותה הרחבה. במסלול הבסיסי נעבוד **ב-Cursor עם Claude Code**; מי שמעדיף VS Code נקי יכול לעשות את אותן צעדים שם."
- **אם לא מספיק:** אין diff בפרויקט — פתחו קובץ טקסט, הוסיפו שורה ידנית, הציגו את **תצוגת השינוי** של העורך (או צילום מסך אחד מהתיעוד הרשמי של ההרחבה).

### S04 | 03:00–04:00 | צ'אט בדפדפן מול סוכן על הפרויקט

- **מה על המסך:** חלון דפדפן (Claude.ai / ChatGPT — ללא מידע רגיש) לצד Cursor עם **Open Folder** על תיקייה בשם ברור (למשל `claude-cursor-practice`).
- **פעולות במסך:** גלילה קצרה בדפדפן על שדה הקלט; ב-Cursor — לחיצה על קובץ אחד ב-Explorer.
- **קריינות:** "בדפדפן מגבילים אותכם בהקשר, בהעלאת קבצים, ובעריכה ישירה של כל הפרויקט. עם Claude Code על **תיקיית פרויקט** הסוכן רואה מה שאתם רואים, יכול לערוך קבצים ולהריץ פקודות במסוף — ואחריות גדולה יותר. לכן נלמד עצירות בטוחות."
- **עצירת כיתה — משימה 1** (אחרי הסצנה).
- **אם לא מספיק:** אין חיבור לאינטרנט — שתי שקופיות מוכנות עם צילומי מסך סטטיים.

### S05 | 04:00–04:50 | חלון הקשר כשולחן עבודה

- **מה על המסך:** גרפיקה פשוטה: שולחן קטן; "כרטיסיות" או "ניירות" נדחפים החוצה כשמספר עולה על המסך (אנימציית 5–10 שניות); אופציונלי: מספר "tokens" מטאפורי בלבד (לא מספרים אמיתיים של מוצר).
- **פעולות במסך:** לולאת אנימציה איטית — לא טקסט קריטי שצריך לקרוא.
- **קריינות:** "תחשבו על חלון ההקשר כשולחן עבודה מוגבל. ככל שהשיחה נמשכת והסוכן קורא יותר קבצים, השולחן מתמלא. בהמשך נלמד לנקות, לקצר, ולשמור החלטות בקבצים — לא רק בצ'אט."
- **אם לא מספיק:** תמונה סטטית אחת + חץ שמסמן "מלא" — בלי אנימציה.

### S06 | 04:50–05:50 | אתם לא "לומדים קודם תחביר"

- **מה על המסך:** תרשים זרימה אופקי: ארבעה תיבות — **דרישות** → **תוכנית** → **אישור** → **שינוי קטן**. ללא קוד.

- **קריינות:** "המטרה כאן לא להפוך אתכם למכונות תחביר. המטרה היא לנסח דרישות, לבקש תוכנית, לאשר שלבים, ולזהות מתי משהו נראה מסוכן — יותר מנצח תזמורת מאשר 'מתכנת קלאסי'."

### S07 | 05:50–06:45 | הסטאק של המסלול

- **קריינות:** "במסלול הבסיסי נשב על **Cursor** כעורך, עם **Claude Code for VS Code** מותקן מהמארקטפלייס. זה משלב את נוחות ה-diff והפרויקט עם הסוכן של Anthropic. בשיעור הבא נתקין ונסייר — איטי ובטוח."
- **אם לא מספיק:** שקופית אחת עם שלושה באלטים — מספיק.
