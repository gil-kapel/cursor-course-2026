# Lesson 3.1 — Voiceover (ElevenLabs / Hebrew)

Plain paragraphs for TTS. Section headers match on-screen beats. Trim pauses in ElevenLabs after timing the cut.

---

## 00:00 – 01:30

MCP הוא פשוט חיבור בין Cursor לבין כלי חיצוני. זה יכול להיות Stitch, Figma, GitHub, דפדפן, או מקור מידע אחר.

מבחינה מעשית, זה אומר דבר אחד: במקום להעתיק מידע ידנית, ה-Agent יכול להביא אותו ישירות לתוך המשימה שלכם.

---

## 01:30 – 03:00

איך בוחרים MCP? לא מתחילים מהכלי. מתחילים מהמשימה.

אם אתם רוצים לבנות UI מהר יותר, Stitch MCP יכול לעזור. אם אתם עובדים מול עיצוב קיים, Figma MCP יכול לעזור. אם אתם עובדים מול issues או PRs, GitHub MCP יכול להתאים. אם אתם צריכים מידע עדכני, Web Search או Fetch יהיו בחירה טובה.

הכלל הפשוט הוא: **בעיה אחת ברורה, MCP אחד מתאים**.

---

## 03:00 – 04:30

הזרימה עצמה קצרה מאוד:
מחברים את ה-MCP בהגדרות.
בודקים שהוא זמין.
מריצים פעולה אחת קטנה.
ואז מבקשים פלט שימושי.

פלט שימושי יכול להיות סיכום קצר, רשימת משימות, פירוק של מסך לקומפוננטות, או תשובה מדויקת מדוקומנטציה. אם יצא לכם רק "הוא הצליח לקרוא", זה עדיין דמו. אם יצא משהו שאפשר לעבוד איתו, זו כבר אינטגרציה אמיתית.

---

## 04:30 – 06:00

לכן התרגיל בשיעור הזה יהיה תמיד בשני שלבים: קודם להשתמש ב-MCP על משימה אמיתית, ואז לשמור ממנו תוצאה אחת בתוך הפרויקט.

התוצאה הזו יכולה להיות notes, task list, UI breakdown, או החלטה טכנית קצרה.

ה-checklist שלכם לכל MCP חדש הוא פשוט:
מה המשימה?
איזה MCP מתאים?
האם בדקתי שהחיבור עובד?
מה הפלט שאני רוצה לקבל?
ואיפה הוא נכנס בפרויקט?

זה כל הסיפור. מתחילים בקטן: MCP אחד, משימה אחת, תוצאה אחת.
