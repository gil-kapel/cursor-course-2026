# Script Master: B09 Project A — Personal Automation (Data / Files)

## Metadata

- Lesson ID: B09
- Title: פרויקט א': אוטומציה אישית (נתונים / קבצים)
- Target duration: 10:00
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

לבנות סקריפט קטן שמעבד CSV (או קבצים דומים) עם ליווי Agent; כשיש שגיאה — להדביק לוג ל-Ask/Debug בלי להבין כל שורה.

## Examples (דוגמאות קונקרטיות)

### רעיונות טובים לאוטומציה ראשונה

- סיכום הוצאות מתוך `sample.csv`
- מיון רשימת לידים לפי סטטוס
- הפקת `summary.txt` מתוך קובץ נתונים פשוט

### קלט ופלט שאפשר להסביר בקלות

- קלט: `sample.csv`
- פלט: `summary.txt`
- בדיקה: האם נוצר קובץ, והאם מספר אחד או שורה אחת נראים נכונים

### סימני אזהרה של Scope

- הסוכן מציע פתאום מסד נתונים
- הסוכן רוצה להקים שרת
- הסוכן מוסיף יותר מדי ספריות בשביל תוצר קטן

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** לשמור את הפרויקט קטן ומקומי. לא להפוך את שיעור האוטומציה לשיעור ארכיטקטורה.

### משימה 1 — להגדיר קלט ופלט (אחרי S02)

- **עצירה:** אחרי חיבור ל-PRD.
- **מה לעשות:** לכתוב בשתי שורות מהו קובץ הקלט ומהו קובץ או סוג הפלט שאתם מצפים לו.
- **איך לוודא הצלחה:** ההגדרה מספיק ברורה כדי לבדוק בסוף אם הפרויקט הצליח.

### משימה 2 — להריץ ולבדוק תוצר (אחרי S05)

- **עצירה:** אחרי הרצת הסקריפט.
- **מה לעשות:** להריץ את הסקריפט ולפתוח את קובץ הפלט או את הפלט במסוף.
- **איך לוודא הצלחה:** יש תוצר ממשי, ואתם בודקים לפחות שורה אחת או מספר אחד בעיניים.

### משימה 3 — לטפל בכישלון בלי פאניקה (אחרי S06-S07)

- **עצירה:** אחרי מסלול השגיאה והגבלת ה-scope.
- **מה לעשות:** אם הייתה שגיאה, להדביק אותה ל-Ask/Debug; אם לא הייתה שגיאה, לכתוב מה הייתם עושים אם הפלט לא היה נוצר.
- **איך לוודא הצלחה:** יש לכם צעד הבא ברור, ולא מעבר אוטומטי ל"נבנה מערכת גדולה יותר".
