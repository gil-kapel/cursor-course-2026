# Script Master: B12 GitHub — Remote Backup and First Push

## Metadata

## Lesson Goal

חשבון GitHub, מאגר ראשון, חיבור הפרויקט המקומי ל-remote, ו-push ראשון — כך שהקוד וההיסטוריה נשמרים בענן ונראים ב-GitHub. **ללא** פריסה לספק נפרד בשיעור הזה; זה מגיע ב-B13 עם Vercel.

## Examples (דוגמאות קונקרטיות)

### מה אמור להופיע ב-GitHub אחרי push מוצלח

- רשימת קבצים שמתאימה לפרויקט המקומי
- לפחות commit אחד עם הודעה ברורה
- ענף (לרוב `main`) עם היסטוריה

### טעויות נפוצות ב-push ראשון

- יצירת repo עם README כשדוחפים פרויקט קיים — עלול ליצור קונפליקט; לעקוב אחרי ההנחיות על המסך
- לשכוח לבצע commit מקומי לפני push
- להריץ פקודות Git בלי להבין — חוזרים ל-B03

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** המטרה בשיעור הזה היא **רק** GitHub + push. הפריסה ל-URL חי ב-B13.

### משימה 1 — מאגר GitHub (אחרי S02)

- **עצירה:** אחרי יצירת החשבון וה-repo.
- **מה לעשות:** ליצור repository או לזהות repo קיים לפרויקט התרגול.
- **איך לוודא הצלחה:** יש URL של GitHub שאפשר לפתוח בדפדפן.

### משימה 2 — push ראשון (אחרי S04)

- **עצירה:** אחרי החיבור ל-remote וההסבר מה עושה push.
- **מה לעשות:** לבצע push מהפרויקט המקומי ל-GitHub.
- **איך לוודא הצלחה:** ה-commit והקבצים מופיעים ב-GitHub, לא רק במחשב המקומי.

### משימה 3 — לוודא שרואים את הקוד בענן (אחרי S05)

- **עצירה:** אחרי ההדגמה של בדיקה ב-GitHub.
- **מה לעשות:** לפתוח את ה-repo בדפדפן ולוודא שקובץ אחד לפחות שאתם מכירים מופיע שם.
- **איך לוודא הצלחה:** מישהו אחר יכול לראות את אותו קוד בלי גישה למחשב שלכם (לפי הגדרות ה-repo).

### S01 | 00:00–00:40 | Hook

- **מה על המסך:** אנימציה/שלושה פריימים: תיקייה מקומית → לוגו GitHub → רשימת קבצים בדפדפן.
- **פעולות:** מעבר מהיר; בלי הקלדה.
- **קריינות:** "עד עכשיו הכול היה על המחשב שלכם. GitHub הוא גיבוי ונקודת שיתוף: אחרי push הקוד שלכם נשמר בענן ואפשר להמשיך ממנו לפריסה — בשיעור הבא."
- **אם לא מספיק:** שקופית אחת עם שלושה אייקונים וקריינות קצרה.

### S02 | 00:40–01:50 | Create GitHub Account and Empty Repo

- **מה על המסך:** דפדפן — `github.com` → New repository; דגש על שם ה-repo ועל אפשרות "בלי README" כשדוחפים פרויקט קיים.
- **פעולות:** גלילה על טופס; ללא סיסמאות גלויות; ללא מייל אמיתי בזום.
- **קריינות:** "יוצרים חשבון אם אין, פותחים repository. אם דוחפים פרויקט קיים, בוחרים לפי ההנחיות על המסך — לפעמים בלי README ראשוני כדי לא ליצור התנגשות."
- **עצירת כיתה:** **משימה 1** — מאגר GitHub.
- **אם לא מספיק:** צילום מסך מטושטש של מסך "ריק" + הסבר קולי; עצירת כיתה נשארת.

### S03 | 01:50–03:10 | Connect Remote and Push (GUI-First)

- **מה על המסך:** Cursor — Source Control / כפתור Publish / שדה `origin`; או הוראות מהסוכן עם פקודות git (הדגישו: B03 לפני הרצה).
- **פעולות:** לחיצה על Publish או הדבקת `git remote` רק אחרי הסבר; גלילה על רשימת remotes.
- **קריינות:** "מחברים את המאגר המקומי ל-remote. ב-Cursor יש לעיתים Publish או הגדרת `origin`. אם הסוכן מציע פקודות — חוזרים ל-B03 לפני הרצה."
- **אם לא מספיק:** תרשים זרימה: מקומי → remote; בלי פקודות חיות אם הכיתה לא מוכנה.

### S04 | 03:10–04:20 | What Push Does (Plain Language)

- **מה על המסך:** דיאגרמה פשוטה: ענפי commits מקומיים → חץ לענן GitHub; או ממשק היסטוריה ב-Cursor ליד אייקון ענן.
- **פעולות:** דיבור מול דיאגרמה; אופציונלי `git log` חד-שורתי בלי ריצה ארוכה.
- **קריינות:** "Push מעתיק את היסטוריית ה-commits לשרת GitHub. זה גיבוי, שקיפות, ונקודת יציאה לכל מה שיבוא אחר כך — כולל פריסה."
- **עצירת כיתה:** **משימה 2** — push ראשון.
- **אם לא מספיק:** הגדירו push כ"העלאת היסטוריה" במשפט אחד והמשיכו ל-S05 עם צילום מסך מוכן.

### S05 | 04:20–05:30 | Verify on GitHub

- **מה על המסך:** דפדפן — דף ה-repo; טאב Code עם קבצים; לחיצה על commit אחרון.
- **פעולות:** השוואה ויזואלית לשם קובץ מהפרויקט המקומי; רענון עמוד.
- **קריינות:** "נכנסים ל-GitHub ובודקים: הקבצים שם, ההודעה על ה-commit הגיונית. אם משהו חסר — חוזרים ל-Cursor ולא לנחש."
- **עצירת כיתה:** **משימה 3** — לוודא קוד בענן.
- **אם לא מספיק:** הראו repo דמה מוכן; אמרו מה התלמידים צריכים לראות ב-URL שלהם.

### S06 | 05:30–06:40 | Secrets Stay Out of the Repo (Teaser)

- **מה על המסך:** קובץ `.env` עם שם מטושטש + סימון אדום "לא ב-commit"; או `.gitignore`.
- **פעולות:** הצבעה על `.gitignore`; ללא ערכי סוד.
- **קריינות:** "מפתחות API וסיסמאות לא שייכים ל-commit. אם צריך סוד לפרויקט — נגדיר אותו אצל ספק הפריסה. בשיעור הבא, עם Vercel, נראה איפה זה יושב ברמת מתחילים."
- **אם לא מספיק:** משפט אחד על סודות + מעבר ל-S07.

### S07 | 06:40–07:30 | Bridge to B13

- **מה על המסך:** לוגו Vercel קטן + טקסט "B13"; חץ מ-GitHub ל-Vercel.
- **פעולות:** שקופית; בלי התחברות ל-Vercel כאן.
- **קריינות:** "עכשיו שהקוד ב-GitHub, אפשר לחבר את אותו repo ל-Vercel ולקבל כתובת חיה. זה השיעור הבא — אל תדלגו עליו אם אתם רוצים URL אמיתי."
- **אם לא מספיק:** 30 שניות; רק "GitHub → Vercel בשיעור הבא".
