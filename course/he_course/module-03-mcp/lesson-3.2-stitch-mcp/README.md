## Syllabus topic

How to use Stitch MCP at the start of the dev step to get unstuck faster, generate a first direction for one screen section, and turn that output into a real component plan or first implementation inside the app.

## Practical outcomes

- Use Stitch MCP when starting from a blank or weak frontend section.
- Generate one better first direction for a real screen section.
- Turn Stitch output into a small implementation plan instead of copying generated code blindly.
- Build or improve one real section in the app using the output as a starting point.
- Understand where Stitch helps most: speed, momentum, and first draft quality.

## Student task

- Pick one weak or missing section in your app, such as a hero, pricing block, feature grid, or settings form.
- Use Stitch MCP to generate one improved direction for that section.
- Choose only one version or idea from the output.
- Turn it into either one implemented section or a short UI task list for the next build step.
- Verify the result looks better and fits the existing app.

## Prompt

`You are a frontend builder. Goal: use Stitch MCP to create a stronger first version of one section in my app. Context: I am working on [screen/section]. Output: 1 recommended direction, a short list of components to build, and the simplest next implementation step. Constraints: keep it beginner-friendly, reuse existing app patterns, and do not rebuild the whole page.`
