## Syllabus topic

Practical risk review: auth assumptions, permissions, secrets handling, data exposure, and abuse cases before implementation hardens the wrong things.

## Learning outcomes

- Run a **structured security pass** focused on realistic threats (auth, permissions, secrets, data leaks, abuse) rather than checklist theater.
- Capture findings that inform **implementation** (2.7) and **code review** (2.8).

## Where this sits

- **Phase:** Planning gate — ideally after spec + architecture exist, before or alongside first serious coding.
- **Prerequisite:** Enough context to review — typically outputs from lessons **2.2–2.5** (PRD through UI plan), plus any early technical notes from **2.3**.
