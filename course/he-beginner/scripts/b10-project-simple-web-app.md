# Script Master: B10 Project B — Simple Web App + Visual Iteration

## Metadata

- Lesson ID: B10
- Title: פרויקט ב': אפליקציית רשת פשוטה + איטרציה חזותית
- Target duration: 10:30
- Language: Hebrew
- Format: avatar narration + screen recording
- Track: beginner

## Lesson Goal

לבנות דף או אפליקציית רשת מינימלית (HTML/CSS/JS או מסגרת פשוטה); לולאת משוב עם Plan לפני שינויי UI גדולים; צילום מסך לצ'אט אם המוצר תומך.

## Examples (דוגמאות קונקרטיות)

### רעיונות טובים לפרויקט ווב ראשון

- דף נחיתה אישי עם כותרת וכפתור
- רשימת משימות קטנה עם פעולה אחת
- דף לינקים פשוט עם מצב בהיר/כהה

### פידבק ויזואלי קצר שאפשר לתת ל-Cursor

- "הכותרת גדולה מדי."
- "הכפתור קרוב מדי לשוליים."
- "המרווח בין הכרטיסים קטן מדי."

### גבולות טובים לאיטרציה הראשונה

- מסך אחד
- אלמנט אינטראקטיבי אחד
- בלי auth, backend, או database

## User tasks — pause and verify (עצירות בכיתה)

**הנחיה למנחה:** לשמור על תחושת ניצחון מהירה. עדיף דף קטן שעובד מאשר פרויקט גדול שלא נפתח.

### משימה 1 — מתרגמים סקיצה לקבצים (אחרי S02)

- **עצירה:** אחרי המעבר מ-ASCII לקבצים.
- **מה לעשות:** לכתוב אילו 2-4 קבצים נדרשים לגרסה הראשונה.
- **איך לוודא הצלחה:** רשימת הקבצים קצרה וברורה, ולא מתחילה לגדול לפרויקט מורכב.

### משימה 2 — לפתוח בדפדפן ולבדוק פעולה אחת (אחרי S03)

- **עצירה:** אחרי פתיחת הפרויקט בדפדפן.
- **מה לעשות:** לפתוח את הדף ולוודא שאלמנט אחד לפחות מופיע או מגיב.
- **איך לוודא הצלחה:** אתם רואים מסך עובד, לא רק קבצים בעורך.

### משימה 3 — תיקון UI קטן ושמירה יציבה (אחרי S04-S07)

- **עצירה:** אחרי לולאת הפידבק החזותי, ה-Plan הקצר, וההמלצה לבצע commit.
- **מה לעשות:** לבקש תיקון UI קטן אחד בלבד, וכשהמסך נראה סביר ועובד לבצע commit.
- **איך לוודא הצלחה:** הבקשה ממוקדת לשינוי קטן אחד, ויש נקודת שמירה שאפשר לחזור אליה לפני עוד ניסויים.
